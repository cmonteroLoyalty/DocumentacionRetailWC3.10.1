Card Management v3.4.1

New features

The new features in this release are:

  • Web widgets

    • The user can now request the PIN to be delivered by post or reset their PIN within their online experience.

  • Native mobile widgets

    • The user can now set and manage limits of withdrawal, online payment and terminal purchase for their debit or credit card.

Messages v3.4.1

New features

  • While messaging with the representative of the financial institution, the widget hides the name of the employee from the customer. By default, the name that is visible to the customer is Customer Service. This can now be replaced by a custom employee name.

Notifications v3.4.1

New features

The new features in this release are:

  • Notification Badge widget - new in Widget Collection Retail Banking.

    • See the number of unread notifications

    • View a list of unread notifications

    • Read notifications in a detailed view

    • Load more of the older notifications

Payments v3.4.1.3

The changes in this release are:

  • The required validation has been set for the payment schedule in the Initiate Payment widget.

Bug fixes

The following issues have been fixed in this release:

  • DBSA-5102 - Initiate Payment widget - cannot review payment after editing.

To update your Retail Banking Widget Collection to v3.4.1-sp.3, run the following command from your project application directory:

npm i --save @backbase/retail-ang@3.4.1-sp.3

User Profile v3.4.1

The highlights for this release are:

  • User Profile client - new client in Widget Collection Retail Banking.

  • User Profile Overview widget - new widget Widget Collection Retail Banking.

New features

The new features in this release are:

  • Native mobile clients

    • User Profile client

      • New client to fetch your name, email addresses, postal addresses, and phone numbers.

      • For Android: This our first client written fully in Kotlin. This means you must include some additional dependencies in your project. If you integrate the client using our Maven repository, use the following line in your build.gradle to let the build system pull your dependencies automatically:

        implementation "com.backbase.android.clients:user-profile-client:4.1.0"

        If you integrate the client manually, include all the dependencies in your build.gradle as listed in the POM reference.

  • Native mobile widgets

    • User Profile Overview widget

      • New widget to view your name, email addresses, postal addresses, and phone numbers.

      • For Android: This our first widget written fully in Kotlin. This means you must include some additional dependencies in your project. If you integrate the client using our Maven repository, use the following line in your build.gradle to let the build system pull your dependencies automatically:

        implementation "com.backbase.android.widgets:user-profile-widget:4.1.0"

        If you integrate the widget manually, include all the dependencies in your build.gradle as listed in the POM reference.

Cross-capability changes

This release note lists the changes for in this release that affect several or all capabilities.

New features

The new features in this release are:

  • For Android widgets, it is no longer necessary to add the @aar extension to widget and client dependencies in the build.gradle file. For example, to add the Bill Pay client and Bill Pay widget dependencies, add the following lines in your build.gradle:

    implementation "com.backbase.android.clients:bill-pay-client:4.1.0"
implementation "com.backbase.android.clients:bill-pay-widget:4.1.0"

    Following this recommendation it is no longer necessary to include transitive Android widget and client dependencies as part of the dependency of your app (such as common-utils or common-ui). They are automatically pulled by Gradle.
