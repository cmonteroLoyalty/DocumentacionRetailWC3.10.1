Web widgets

The web widgets included in Transactions are:

  • View transactions list - view the list of transactions for a single account.

View transactions list

Bank customers use the View transactions list widget to view the list of transactions for a single or multiple accounts.

Features

This widget provides the following features for bank customers:

  • View the list of transactions for a single or multiple accounts. By default, transactions are grouped by booking date.

  • View the details of a specific transaction. To view details about a transaction, the bank customer clicks the transaction.

  • Dispute or inquire about a transaction.

    Bank customers can dispute a transaction they disagree with, or inquire about a transaction by sending a secure message to their bank. This enables the bank customer to send a dispute request directly, and easily contact customer service. To initiate a dispute or an inquiry, the bank customer selects the Report button in the transaction details.

It is possible to send an inquiry for any transaction. For disputes, the transaction is validated first by the following criteria:

  • The billing status of the transaction is disputable. By default, pending and unbilled transactions are disputable. Developers can configure the billing statuses that transactions can have to be disputable.

  • The transaction is eligible for a dispute based on the eligibility period. Developers can configure the eligibility period. The default value is 120 days from the transaction date.

  • The transaction type. Developers can configure the disputable transaction types. By default, credit card and debit card transactions are disputable.

    • View the account number associated with the transaction, if the bank customer selected more than one account.

    • View images for cleared checks, if the transaction is a check withdrawal. When the bank customer hovers over a check image with the pointer, an eye icon with a grey overlay is displayed on the image, indicating that the image can be viewed in a larger size. Once the image is enlarged, the bank customer can download it. The supported formats are JPEG and PNG. The name of the downloaded file specifies the check number and which side of the check the image depicts. For example: 2061-back.png. If the image is damaged or there is an internal error in the system that provides it, an information tooltip is displayed detailing the cause of the error.

    • Search the transactions list using keywords. The search action takes place once the bank customer clicks the magnifier icon, or the enter key in the keyboard after some letters or numbers are entered to the search box.

    • Filter the transactions list by the following criteria:

  • Credit/debit indicator and transaction type: Dropdown menus where only one item can be selected for each criterion.

  • Date, amount, check serial number, and billing status: From/To input fields, where the bank customer specifies a range by filling in either one, or both of the fields.

    • Print the transactions overview.

    • Add, modify, view, and delete notes for each transaction.

    • Categorize transactions.

    • Export all transactions, or a list of transactions that is the result of applying filters or search.

    • Sort transactions. By default, the most recent transactions is on the top of the list.

    • View the geolocation of each transaction in a map indicated by a marker. The full open address of the transaction is listed together with the map if the data is available. If the bank customer clicks the marker, they get redirected to a navigation application, for example Google Maps, where the address of the transaction is prefilled.

The following preferences of this widget are configurable for administrators in Experience Manager:

  • Configure the way transactions are loaded:

    • Pagination: An index of available pages is displayed under the list, which always provides links to the first and the last pages, as well as to a number of pages adjacent to the current one.

    • Load more: If the bank customer clicks the Load more button under the list, additional transactions are loaded. The button remains visible until there are no more items to view.

  • Configure the transactions list based on transaction status:

    • List pending transactions on top of completed transactions, and hide the billing status filter.

    • Group transactions based on their booking date value, regardless of their billing status value. Transactions with billing status Pending or Unbilled are indicated with a pill.

  • Configure the number of transaction records shown per page, and the number of navigation pages.

  • Configure the default sorting field and direction.

  • Configure whether the bank customer can change the category of a transaction.

  • Configure which filters are visible to the bank customer.

  • Configure whether check images are visible in the transactions details.
