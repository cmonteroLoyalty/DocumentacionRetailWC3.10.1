Retail Banking Widget Collection v3.1.4

The highlights for this release are:

  • Web widgets

    • (new) Messages: Messages Conversations List widget.

    • Retail Banking Experience: a newly designed RB Experience is now available.

    • Payments: the Initiate Payment widget now supports model extensions.

  • Native mobile widgets

    • (new) Card Management: view an overview of credit/debit cards.

    • (new) Card Management: temporarily lock/unlock a credit/debit card.

    • (updated) Bill Pay: now includes electronic payee methods.

See the individual release notes for each capability:

  • Access Control - no changes.

  • Account Statements - no changes.

  • Authentication - no changes.

Bill Pay v3.1.4

This release note lists the changes for the Bill Pay capability in this release.

New features

The new features in this release are:

  • Native mobile widgets

    • Electronic Payee feature added. For more information see the Bill Pay Widget documentation.

Breaking changes

The following breaking changes have been introduced in this release:

  • Native mobile widgets

    • Android Only - The bb.action.bill-pay.pay-payee event payload has been modified to conform with the expected payload structure for the same event in Bill Pay Form Widget. Previously the event published a single PayeeSummary object but now publishes an array containing at least one PayeeSummary object in the following format:

      {
  "key_payee_list":
  [
    {
      "id": "testID",
      "name": "testName",
      "electronic": true,
      "paymentServices": []
    }
  ]
}

Card Management v3.1.4

This release note lists the changes for the Card Management capability in this release.

New features

The new features in this release are:

  • Native mobile widgets

    • Card Management native widget is released with the following features:

      • Ability to view the overview of credit/debit cards

      • Ability to lock temporarily and unlock a credit/debit card

  • Native mobile clients

    • Card Management native client is released with the following features:

      • Retrieve the list of cards associated with the user

      • Retrieve a specific card of user based on card id

      • Change the status of the card to lock/unlock

Places v3.1.4

This release note lists the changes for the Places capability in this release.

Known issues

  • Native mobile

    • MAINT-8626: App Store rejection due to use of non-public API in Places widget for iOS.

      The Places widget for iOS version has a blocker for App Store release. For details, visit the Community.

      This issue was fixed in Widget Collection Retail Banking v3.2.1. Upgrade to this version. When upgrading to v3.2.1, consider the following:

      Native mobile widgets have a new technical version number as of Widget Collection Retail Banking v3.2.0. Until v3.2.0, the widget collection has followed the version format <release>.<major>.<minor>.<patch>. As of the v3.2.0 major release, the <release> digit has been removed from artifacts. This means that, from that release on, the native mobile widgets are versioned as v2.1.0 in the Backbase repository instead of 3.2.1(.0). This change was introduced because customers with patched versions of the collection, for example 3.2.1.1, were unable to publish apps to the App store. This is due to the App Store Connect rules around apps with 4-digit version numbers. The <release> number is now part of the repository URL, which is the version number developers add to their local CocoaPods and Gradle installations. This affects both for Android and iOS.

      To use the new major version of Widget Collection Retail Banking, update the Gradle and Cocoapods dependencies to reflect the repository name change. It is important to understand that the actual product version is still 3.2.1(.0).

      For example:

      pod 'PlacesWidget', '2.1.0'
pod 'TransactionsWidget', '2.1.0'

Products v3.1.4

This release note lists the changes for the Products capability in in this release.

New features

The new features in this release are:

  • Account Selector: accounts listed here should give min details per account.

    • Following the design rules, a requirement od minimum UI/UX sections have to be shown to a User, through usage of default Product Kind card component "bb-product-item-basic-account-ui".

  • Updated Product Details widget in WC3 to Angular 6, retail collection, according to bb atomic design, by the latest UX designs.

  • Add "total" label in front of the Product Kind group "Aggregated Balance"

    • Changes were made in customizable header template bbProductSummaryHeaderCustomizable where we change the HTML structure so we can achieve the same UX/UI as in design.
