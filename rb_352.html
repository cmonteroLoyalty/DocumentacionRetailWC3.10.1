Bill Pay v3.5.2

This release note explains the changes to this capability.

New features

The new feature in this release is:

  • Bill Pay Payee Summary widget

    The new Bill Pay Payee Summary widget enables a Bill Pay customer to see a summary view containing the payee details. It also introduces a way to quickly navigate between payees.

  • New native Bill Pay Schedule widget

    The new Bill Pay Schedule widget enables a Bill Pay customer to schedule a payment and set the payment frequency. It also enables the customer to select the payment date, including the option to choose the earliest free delivery date, rush date, or other date.

Bug fixes

The following issue has been fixed in this release:

  • MAINT-8748: Trying to extend the Bill Pay Payments widget using the bbBillpayPaymentsListCustomizable extension slot produced the error No Provider for BillPayPaymentsRouterService.

Deprecations

The following features are deprecated from this release, and support for these features ceases in v3.7.0:

  • editPayment has been deprecated from the Bill Pay Manage Payments widget. This is replaced by isEditPayment and paymentType.

  • The following methods, events, and fields have been deprecated from the Bill Pay Form widget:

    • Android

      • The method getPaymentRequest() from BillPayFormContract is deprecated. Use updateViewWithPaymentRequest instead.

      • The method selectDate from BillPayFormContract is deprecated. Use selectScheduleInformation instead.

      • The method paymentScheduleFinished from BillPayFormContract is deprecated without an alternative. This method should no longer be used.

      • The method updateViewWithDate from BillPayFormView is deprecated. Use BillPayScheduleWidget instead.

      • The event BillPayFormWidget.EVENT_PAYEE_REQUESTED is deprecated. Use BillPayFormEvents.EVENT_PAYEE_REQUESTED instead.

      • The event BillPayFormWidget.EVENT_BILL_PAY_REVIEW is deprecated. Use BillPayFormEvents.ACTION_BILL_PAY_SHOW_REVIEW instead.

      • The event BillPayFormWidget.EVENT_BILL_PAY_PAYMENT_SUCCEED is deprecated. Use BillPayFormEvents.ACTION_BILL_PAY_OVERVIEW instead.

      • The event BillPayFormWidget.EVENT_BILL_PAY_PAYMENT_FINISHED is deprecated without an alternative. This event should no longer be used.

      • The event BillPayFormWidget.EVENT_BILL_PAY_FORM_SELECT_DATE is deprecated. Use BillPayFormEvents.EVENT_BILL_PAY_FORM_SELECT_SCHEDULE instead.

      • The event BillPayFormWidget.EVENT_BILL_PAY_FORM_DATE_SELECTED is deprecated. Use BillPayFormEvents.EVENT_BILL_PAY_SCHEDULE_INFO_SELECTED instead.

      • The fields EXPEDITION_PAYMENT, REGULAR_PAYMENT, and OVER_NIGHT_PAYMENT from BillPayFormWidget are deprecated. Use PaymentServiceType enum instead.

    • iOS

      • The method preSelectedDate from BillPayFormContract is deprecated. Use BillPayScheduleWidget instead.

      • The method updateViewWithDate from BillPayFormView is deprecated. Use BillPayScheduleWidget instead.

      • The event paymentProductSelectedEvent from BillPayFormWidget is deprecated. Use BillPayFormEvents.paymentProductSelectedEvent instead.

      • The event paymentSelectRecipientAction from BillPayFormWidget is deprecated. Use BillPayFormEvents.paymentSelectRecipientAction instead.

      • The event paymentCompletedEvent from BillPayFormWidget is deprecated. Use BillPayFormEvents.paymentOverviewAction instead.

      • The event paymentReviewAction from BillPayFormWidget is deprecated. Use BillPayFormEvents.paymentShowReviewAction instead.

      • The event paymentFinishAction from BillPayFormWidget is deprecated. Use your own custom action instead.

      • The event paymentDateSelectedEvent from BillPayFormWidget is deprecated. Use BillPayFormEvents.paymentScheduleSelectedEvent instead.

      • The event paymentDateSelectRequestedAction from BillPayFormWidget is deprecated. Use BillPayFormEvents.scheduleSelectRequestedAction instead.

To update an existing deployment to accommodate these deprecations, see Migration.

Known issues

The known issue in this release is:

  • Introduced in v3.5.2

    • OBPAY-2212: The Bill Pay Payee Summary widget returns a blank error page for all errors.

    • OBPAY-2215: The Bill Pay Payee Summary widget preference managePayee does not exist by default and needs to be added manually.

Payments v3.5.2

This release note explains the changes to this capability.

New features

The new features in this release are:

  • Submit a request to stop an issued check (web)

    The Stop Checks List widget enables a bank customer to submit a request to the bank to stop the clearance of a check (or cheque). This feature is helpful in the following scenarios:

    • The customer has insufficient funds and wants to prevent the check from bouncing.

    • The customer entered the incorrect amount or other details on the check.

    • The customer’s check or check book was lost or stolen. In the case of multiple checks or an entire check book, the customer can stop all checks that fall within a range of serial numbers.

    • The customer mailed the check to the wrong address or it was lost in the mail.

    • The customer has a dispute over purchases.

  • Remove the connection to an external account (web)

    In order to initiate an external account to account (A2A) transfer in the US, the bank customer needs to connect and activate their external account. An external account is an account that the customer holds at a different financial institution.

    The Connect External Account widget now enables the customer to remove the connection to an external account, irrespective of its status. This includes accounts with statuses like Pending Activation or Failed Verification. For example, the customer can remove the connection to an external account if it failed to activate or has been closed.

    Once the connection has been successfully removed, the external account no longer appears in the list of connected accounts.

Products v3.5.2

This release note explains the changes to this capability.

New features

The new features in this release are:

  • Apply arrangement state indicator in Account Details, My Accounts, and Account Selector

    The default retail account statuses - active, inactive, and closed - are now displayed along with an icon in the Account Selector, My Accounts, and Account Details. If custom account statuses are sent, the widgets display the status as is, without an icon.

Bug fixes

The following issues have been fixed in this release:

  • PRSUM-5198: In the Manage Accounts widget, the user was not able to inline edit the account alias if a single word value was introduced. Also, for long aliases that were close to the maximum length of 50 characters, the inline edit pen overlapped the account name.

  • PRSUM-5441 - The visible account property was set to false in the Account Selector widget. If the bank did not send any visible property, accounts were not visible to the bank customer at all.

Transactions v3.5.2

This release note explains the changes to this capability.

New features

The new features in this release are:

  • Display pending transactions on top

    The following widgets now provide the ability to list pending and unbilled transactions in a separate container on top of the transactions list:

    • Transactions List widget (web)

    • Transactions widget (native mobile)

    To configure this feature, use the showPendingTransactionsOnTop preference.

    However, if you want to display pending transactions on top, ensure that the billingStatus column does not have NULL values. When you set showPendingTransactionsOnTop to true, the transactions list widget only displays transactions where the billingStatus is not NULL.

  • Overview of turnover (native mobile)

    The native mobile Transactions widget now provides the ability to view the turnover (incoming and outgoing transactions) for the last six months.

Migration

When you migrate from one version of this product to another, best practice is to update incrementally through all minor and major versions in between.

Follow these steps to update an existing deployment:

  1. Deprecated date events in the Bill Pay Form widget

    Make the following changes to the Bill Pay client and Bill Pay Form widget:

    • Bill Pay client changes

      • Android

        From To

        RecurringPayment.setNumberOfInstances(int numberOfInstances)

        RecurringPayment.setNumberOfInstances(@Nullable Integer numberOfInstances)

        RecurringPayment.getNumberOfInstances()

        RecurringPayment.getNullableNumberOfInstances()

      • iOS

        In RecurringPayment class recurring property is removed due to DBS removal.

    • Bill Pay Form widget changes

      • Bill Pay Form contract

        • Android

          From To

          getPaymentRequest

          updateViewWithPaymentRequest

          selectDate

          selectScheduleInformation()

          selectDate

          selectScheduleInformation(amount, memo)

          getPayeeSummary returns @Nonnull PayeeSummary

          getPayeeSummary returns @Nullable PayeeSummary

        • iOS

          From To

          selectDate

          selectScheduleInformation()

          selectDate

          selectScheduleInformation(amount, memo)

          payeeSummary returns nonnull PayeeSummary

          getPayeeSummary returns nullable PayeeSummary

      • Bill Pay Form view

        From To

        updateViewWithDate

        BillPayScheduleWidget

      • Pub/sub events and properties

        • Android

          From To

          BillPayFormWidget.EVENT_BILL_PAY_REVIEW with bb.action.bill-pay-form.review key

          BillPayFormEvents.ACTION_BILL_PAY_SHOW_REVIEW with bb.action.bill-pay-form.show-review key

          BillPayFormWidget.EVENT_BILL_PAY_PAYMENT_SUCCEED with bb.event.bill-pay-form.success key

          BillPayFormEvents.ACTION_BILL_PAY_OVERVIEW with bb.event.bill-pay-form.overview key

          BillPayFormWidget.EVENT_BILL_PAY_PAYMENT_FINISHED with bb.action.bill-pay-form.finish key

          No replacement

          BillPayFormWidget.EVENT_BILL_PAY_FORM_SELECT_DATE with bb.action.bill-pay-form.select-date key

          BillPayFormEvents.EVENT_BILL_PAY_FORM_SELECT_SCHEDULE with bb.event.bill-pay-form.select-schedule key

          BillPayFormWidget.EVENT_BILL_PAY_FORM_DATE_SELECTED with bb.event.bill-pay-form.date-selected key

          BillPayFormEvents.EVENT_BILL_PAY_SCHEDULE_INFO_SELECTED with bb.action.bill-pay-form.schedule-selected key

          BillPayFormWidget.EXPEDITION_PAYMENT

          PaymentServiceType.EXPEDITED_PAYMENT

          BillPayFormWidget.OVER_NIGHT_PAYMENT

          PaymentServiceType.OVERNIGHT_CHECK

          BillPayFormWidget.REGULAR_PAYMENT

          PaymentServiceType.REGULAR_PAYMENT

        • iOS

          From To

          confirmModeKey with confirm key

          overviewModeKey with overview key

          paymentProductSelectedEvent with bb.event.bill-pay-form.product-selected key

          BillPayFormEvents.paymentProductSelectedEvent with bb.event.bill-pay-form.product-selected key

          paymentSelectRecipientAction with bb.action.selector.select-recipient key

          BillPayFormEvents.paymentSelectRecipientAction with bb.action.selector.select-recipient key

          BillPayFormEvents.paymentOverviewAction with bb.event.bill-pay-form.overview key

          BillPayFormEvents.paymentShowReviewAction with bb.action.bill-pay-form.show-review key

          paymentFinishAction with bb.action.bill-pay-form.finish key

          No replacement

          paymentDateSelectedEvent with bb.event.bill-pay-form.date-selected key

          BillPayFormEvents.paymentScheduleSelectedEvent with bb.action.bill-pay-form.schedule-selected key

          paymentDateSelectRequestedAction with bb.action.bill-pay-form.select-date key

          BillPayFormEvents.scheduleSelectRequestedAction with bb.event.bill-pay-form.select-schedule key

  2. Deprecated the editPayment property in the Bill Pay Manage Payments widget

    Make the following changes to the Bill Pay Manage Payments widget:

    • Bill Pay Manage Payments widget changes

      From To

      editPayment: PaymentType

      isEditPayment: boolean and paymentType: PaymentType

  3. Migration of Messages to the Communication API

    To enable Messages widgets to use the new Communication API, add a feature flag in your app.module:

    BackbaseCoreModule.forRoot({
  logDeprecations: true,
  features: {
    NEW_COMMUNICATION_API_ENABLED: true,
  },
})

    Find more information about this configuration in Communication between widgets.

  4. Migration of the Notification Settings widget to the Communication API

    Configure new output and input properties in the source and destination widgets as described in Communication between widgets.

    For example, to set up communication between the Product Summary List widget and the Notification Settings widget, take the following steps:

    1. In Product Summary List, change the old output.selectedAccount: navigation:product-notifications-settings property to the new output.selectedAccount: route-param notificationsSettingsAccountId property.

    2. In Notification Settings, configure the new input.selectedAccountId: route-param notificationsSettingsAccountId property.
