Web widgets

Understand Messages describes the functionalities of Messages. The following web widgets deliver these functionalities:

  • Send and receive messages

    • Compose message by the customer: Compose and send a message to a bank employee

    • View messages by the customer: View all sent and inbox messages

    • View conversation thread by the customer: View all messages from a single conversation

Compose message by the customer

In the Compose message by the customer widget, bank customers interact with their bank in a secure environment.

The required fields include Topic, Subject, and Body. The customer can also attach a file, save the message as a draft, or discard it. If the message is saved as a draft, it then appears in the Drafts tab in the View messages by the customer widget.

This widget supports the following features of rich text:

  • Bold

  • Italic

  • Underline

  • Strike-through

  • Paragraph

  • Ordered list

  • Unordered list

  • Code

  • Link, including deep links

This widget interfaces with the Messages DBS capability using the message-client-api.

Features

  • Select message topic.

  • Specify message subject.

  • Compose message body.

  • Attach a file to the message.

  • Delete an attachment before sending.

  • Discard a message before sending.

  • Save a message as draft.

  • Send a message.

View messages by the customer

In this widget, the bank customer sees their incoming messages and conversations at a glance.

The following information is listed for each message or conversation:

  • Importance: Specifies the importance of the root message.

  • From: Specifies the human-readable sender name or label.

  • Draft: Specifies whether the conversation has a draft attached to it.

  • Attachment: Specifies whether the conversation has files attached.

  • Subject: Specifies the subject of the root message.

  • Topic: Specifies the topic that the message concerns, for example Savings. The user selects this from a list of available topics.

  • Date: Specifies the date of last received message in the conversation.

The bank customer loads more conversations by clicking Load more. Unread messages are highlighted.

To refresh the list of conversations, the bank customer clicks the refresh icon.

When clicking a conversation in the list, the customer accesses the View conversation thread by the customer widget.

This widget interfaces with the Messages DBS capability using the message-client-api.

Features

This widget provides the following features for the bank customer:

  • Mark conversations as read or unread.

  • Delete conversations that were not marked non-deletable by the sender.

  • View information about a conversation, including sender, subject, importance, topic, and date.

  • View Inbox, Sent, and Drafts separately.

  • Access the View conversation thread by the customer widget.

  • Load more conversations.

  • Refresh the list.

The widget polls the backend service to retrieve the number of unread conversations. The Navigation SPA widget displays this counter as a badge next to Inbox.

For further information, see:

  • Reference - View messages by the customer

View conversation thread by the customer

In the View conversation thread by the customer widget, the bank customer views a single conversation thread and replies to a message in that conversation, if possible.

The bank customer accesses this widget by clicking a list item in the View messages by the customer widget. Once in a conversation thread, individual messages can be opened by clicking them.

The bank customer replies to the conversation by clicking Reply. It is possible to send multiple replies to the same message. The Reply button is not available if the message was marked as read-only by the sender or if the message is a mailout.

A text input field is available with text markup controls. By default, the input is limited to 300 characters. If necessary, the bank customer attaches files to the message.

The bank customer saves the message by clicking Save as draft or discards their message by clicking Discard. If the message is saved as a draft, it then appears in the Drafts tab in the conversation list as well as attached to the conversation when the customer reopens the conversation.

This widget interfaces with the Messages DBS capability using the message-client-api.

Features

  • View a single conversation thread.

  • Download an attachment from an incoming message.

  • Send a reply to the conversation unless it is a read-only message or a mailout.

  • Attach a file to the reply.

  • Delete the attachment before sending.

  • Mark the conversation as read or unread.

  • Delete the conversation unless it was marked non-deletable by the sender.

  • Save the message as draft.
