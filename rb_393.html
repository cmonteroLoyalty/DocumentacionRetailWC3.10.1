Account Statements v3.9.3

This release note explains changes to this capability.

Bug fixes

This release includes the following bug fixes:

  • Accessibility issue: Screen reader - Link purpose

    Users using a screen reader now have additional information about the account name, book date, and category for preview and download actions for each statement in the View and download retail account statements widget.

  • Accessibility issue: Screen reader - Info & Relationship

    Column header names were added for the Preview & Download options in the View and download retail account statements widget, in order for screen readers to properly read them.

This release note explains changes to this capability.

Bug fixes

This release includes the following bug fixes:

  • MAINT-11755: Bill Pay OpenApi .yml requires additions added to some POJOs

    Generated open API clients are now added to data-ang.

Card Management v3.9.3

This release note explains changes to this capability.

Bug fixes

This release includes the following bug fixes:

  • MAINT-11197: Card Management Travel Notice: Correct status of the card is not reflected when user selects a card while creating a travel notice.

    Card statuses are now correctly reflected in the View travel notice widget.

Notifications v3.9.3

This release note explains changes to this capability.

New features

This release provides the following new features:

  • New displaySettingsButton preference

    A new displaySettingsButton boolean preference has been added to the Notification badge widget. The preference controls whether the notification settings icon is displayed or not. This can be useful for apps where setting up notification preferences is not available. The preference is set to true by default. See reference documentation for details.

  • Show more renamed to See more

    The Show more link text changed to See more in the Notification badge widget.

  • Shade and hover

    The See more link is now underlined when hovered over in the Notification badge widget. Also, the shade is added to the each message in the list on hover for better UX.

Payments v3.9.3

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Web: New beneficiary selector

    The Initiate omni payment and Initiate payment wizard widgets come with a new beneficiary selector. It includes following features:

    • Both the Contacts and Own accounts tabs in the beneficiary selector independently support pagination with scrolling.

    • When all records are loaded, the tab displays End of data.

    • When both the From account and Own account fields have the same account selected, the widget shows an error for the beneficiary.

  • Web: Due date messages

    The due date messages have been added to the Initiate omni payment widget for credit card and loan payments. Bank customers will be aware of the late fees if the payment due date has passed.

  • Web: Pay monthly switch

    A second flavor for the schedule component is now available as a pay monthly switch in the Initiate omni payment widget. The switch provides the payment frequency options of Once and Monthly.

Bug fixes

This release includes the following bug fixes:

  • MAINT-11505: Beneficiary selector component - bug while adding and removing details

    While adding beneficiary details manually and then removing the beneficiary name with the cross button or by deleting, the contacts/own accounts tabs are shown as empty. This is fixed in the new beneficiary selector mentioned in the New features above. Both tabs now show the first set of data in this scenario.

  • MAINT-11703: Currency input does not support locales with thousand separators

    For the locale nl-NL, a dot . is added at every 4th position for an amount of 4 or more digits. Similarly, for en-ZA locale, a space is added at every 4th position. The widget sends these amounts with thousand separators to the service, while it should remove the thousand separators before sending. It was fixed by using a custom separator based on the locale.

  • MAINT-11710: Omni Payment - error message improvements

    The validation error message that is shown on the payment entry form and the review form starts with a non-translatable text 'Payment data error'. This text is made translatable as well as added with a configurable flag isErrorTitleDisplayed to remove it completely. Default value of this flag is true, which shows this text.

  • MAINT-11354: Connect external account - account number field on FE allows for 19chars, BE response allows max of 17

    The account number field on the connect external account form allowed for 19 characters, but the BE response allowed the maximum of 17. This issue has been fixed and now allows the maximum of 17 characters.

  • MAINT-11484: Make account number masking configurable for InitiatorFields.initiatorAccountGroup component

    The InitiatorFields.initiatorAccountGroup component is now configurable.

  • MAINT-11496: Web: currency symbol showing instead of 'VND' (vietnamese) currency for InitiatorFields.initiatorAccountGroup component.

    Developers can now configure the Initiate omni payment widget to show either the currency symbol or the currency code.

Products v3.9.3

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Web: Synchronized backend and frontend account masking

    Rather than consuming unmasked account numbers from the backend and masking them in the frontend, the masking is now done in the backend. The frontend is fully synchronized with the backend and visualizes the account number that the backend returns.

  • Web: Application-level configuration of account number format

    The spacing required for the formatting of the account number is now configurable per account type. Spacing can be set for every N characters.

User Self Service v3.9.3

This release note explains changes to this capability.

New features

This release provides the following new feature:

  • Web: New output for handling the termination of sessions on password change

    The Manage user security widget now emits a widgetStatus event that shows a user has successfully changed their password. This new output makes it possible for other components to handle the password change correctly.

    For example, if the User Manager service is configured to end a user’s active sessions when the user changes their password, ensure the widgets don’t attempt to make any requests to the backend services, and present a way for the user to navigate to the login page to authenticate with their new password.
