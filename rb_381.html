Account Statements v3.8.1

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Web: Prevent multiple user clicks on download statement option

    The download progress of a statement is now visible in the browser for the View and download retail account statements widget in order to prevent the user clicking the download option multiple times while the download process is already initiated.

Authentication v3.8.1

This release note explains changes to this capability.

Bug fixes

This release includes the following bug fix:

  • ID-2733: Remove theme 1 support from all widgets and projects

    An unused module and constant were removed from the Log in user widget.

Payments v3.8.1

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Web: New payment type in the Initiate omni payment widget

    One INTERNAL_TRANSFER payment type is now available for transferring funds between a bank customer’s own accounts in the same financial institution. Use this payment type for transfers between all kinds of internal accounts, such as, checking or savings. This payment type does not allow setting validations for specific To and From account type combinations. Apply your own filtering logic to the To and From account drop-down list.

    See reference documentation for payment type and account drop-down configuration.

  • Web: Warning messages in the Quick transfer widget

    Warning messages have been added for the mandatory fields in the Quick transfer widget.

  • Web: New designs for Payment Review page

    The payment review page of the Initiate omni payment and Payment wizard widgets has the following changes:

    • New card layout and icons for From account and Transfer to account details.

    • Balance of the From account is shown in its card.

    • Bank branch code of beneficiary’s bank is shown in Transfer to card.

    • For international transfers, the flag of the destination country of the payment is shown in Transfer to card.

    • Available details like chosen payment option, transfer fee, fee sharing option are shown.

  • Web: Initiate omni payment config for International Transfers out of beta

    The config for International Transfers in the Initiate omni payment widget is now out of beta and fully supported.

  • Web: Payment wizard out of beta

    The Payment wizard widget is now out of beta and fully supported.

Products v3.8.1

This release note explains changes to this capability.

Bug fixes

This release includes the following bug fixes:

  • MAINT-10251: Edit Account Alias has invalid maxlength in the code

    The maxlength of Edit Account Alias was set to infinity, which meant that the validation didn’t work. We have corrected the maxlength to the proper value so that input validation now works.

  • PRSUM-6968: It was not possible to search on the Account Alias after changing it.

    This has been fixed so that after changing the Account Alias, it is now possible to search on the alias.

CVEs

Backbase products contain open source libraries and frameworks, also known as third-party components. Information on a security vulnerability found in a third-party component is made publicly available. Vulnerabilities are typically assigned an official CVE number by MITRE. There are several publicly accessible vulnerability databases that provide background information on the vulnerability. The CVE records can also include mitigation or exploitation steps.
