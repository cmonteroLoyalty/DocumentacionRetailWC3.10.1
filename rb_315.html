Payments v3.1.5

This release note lists the changes for the Payments capability in this release.

New features

The new features in this release are:

Web widgets

  • Upcoming Payments - new in Retail Banking Widget Collection. This widget supports the following:

    • View payment orders with future date and ACCEPTED status for an account which the user has access to within a selected context.

    • Display details for a payment order upon selection.

    • Load payment orders incrementally.

    • Cancel an upcoming payment that has not been executed yet.

Bug fixes

The following issues have been fixed in this release:

  • Initiate Payment widget - max length of 34 characters for an IBAN has been enforced.

  • Initiate Payment widget - fixed an issue where the invalid field values were prematurely validated. The validations are now triggered when the user proceeds to the review payment screen.

Places v3.1.5

This release note lists the changes for Places in this release.

New features

The new features in this release are:

Web widgets

  • Places widget - new in Retail Banking Widget Collection.

Known issues

Native mobile widgets

  • MAINT-8626: App Store rejection due to use of non-public API in Places widget for iOS.

    The Places widget for iOS has a blocker for App Store release. For details, visit the Community.

    This issue was fixed in Widget Collection Retail Banking v3.2.1. Upgrade to this version. When upgrading to v3.2.1, consider the following:

    Native mobile widgets have a new technical version number as of Widget Collection Retail Banking v3.2.0. Until v3.2.0, the widget collection has followed the version format <release>.<major>.<minor>.<patch>. As of the v3.2.0 major release, the <release> digit has been removed from artifacts. This means that, from that release on, the native mobile widgets are versioned as v2.1.0 in the Backbase repository instead of 3.2.1(.0). This change was introduced because customers with patched versions of the collection, for example 3.2.1.1, were unable to publish apps to the App store. This is due to the App Store Connect rules around apps with 4-digit version numbers. The <release> number is now part of the repository URL, which is the version number developers add to their local CocoaPods and Gradle installations. This affects both for Android and iOS.

    To use the new major version of Widget Collection Retail Banking, update the Gradle and Cocoapods dependencies to reflect the repository name change. It is important to understand that the actual product version is still 3.2.1(.0).

    For example:

    pod 'PlacesWidget', '2.1.0'
pod 'TransactionsWidget', '2.1.0'

Products v3.1.5

This release note lists the changes for the Products capability in in this release.

New features

The new features in this release are:

Web widgets

  • Product Summary Manage Accounts - initial release of ProductSummaryManageAccountsWidgetModule. Add an alias to your arrangements and edit their visibility in the overview. To be able to edit inline text in this widget, the new component bb-input-inline-edit-ui is now added to collection-bb-ui-ang.

  • Product Summary Account Selector - the payload of this widget now returns the Product Type as well as the Product ID.

  • Product Summary List - design changes.

    • Added a separator when the list is collapsed.

    • Removed the blue line in lists (from the UI component and it should be introduced as the active state).

    • Responsive designs are now used on smaller screens.

  • Product Summary View Details - design changes.

    • Increased spacing between sections.

    • Decreased the padding top within sections.

    • Padding between associated card is now 24px.

    • Added Print icon to the print button.

    • Change for size of Product title to 24px (e.g. My Current Account).

    • Changed the font size of the top title and the subtitles of the grouped info to 20px (for example General, Other etc).

    • Tool tip functions based on the guidelines of Backbase Design System.

Transactions v3.1.5

This release note lists the changes for the Transactions capability in this release.

New features

The new features in this release are:

Web widgets

  • Transactions List - a new preference is introduced to show or hide transactions controls such as filter, search, export and print in the widget.

Bug fixes

The following issues have been fixed in this release:

  • MAINT-6741 - Transaction List - PaginationType enum is not exposed.

  • MAINT-6739 - bb-transactions-filter: EnumService is not exposed.

Deprecations

  • TransactionsFilterComponent#close event emitter will be renamed to closeFilter in v3.2.0

  • TransactionsFilterComponent#closeFilter method will be renamed to onCloseFilter in v3.2.0

  • TransactionsFilterComponent#clearFilter method will be renamed to onClearFilter in v3.2.0

  • TransactionsListComponent#PaginationType (which exposes the internal type for pagination type) is deprecated. Two new fields isPaginationPaged and isPaginationLoadMore allow you to get the pagination type without needing the internal type. The deprecated field will be removed in v3.2.0.
