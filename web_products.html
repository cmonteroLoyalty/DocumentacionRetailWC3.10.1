Web widgets

Understand Products describes the functionalities of Products. The following web widgets deliver these functionalities:

List products

  • View retail account list: View of all banking products grouped by product kind

Select products

  • Select retail account: Select an account from a drop-down

Display product details

  • View retail account details: View details of a selected product

Manage products

  • Manage retail account settings: Manage your accounts

Select retail account

The Select retail account widget of the Products capability allows bank customers to select an account from their account list. The drop-down shows the name and the current account balance.

This widget is used in combination with several other widgets. When both widgets are placed on the same page, they can communicate with each other. For example, if the Select retail account widget is placed on the same page as the View transaction list widget, and the bank customer selects an account, the Select retail account widget publishes an event. The View transaction list widget subscribes to this event and changes its view to display the transactions of the selected account.

The widget displays the status of the account along with an icon. The possible statuses are:

  • Active

  • Inactive

  • Closed

This widget interfaces with the Products DBS capability using the Products client API.

Features

The following features are configurable in Experience Manager:

  • Enable or turn off backend account masking.

    • If enabled, the widget requests account masking from the backend service, and if the backend service supports this feature, the backend service sends a masked account number to the widget. DBS supports this feature, see this page for more information.

    • If turned off, the widget does not request account masking from the backend service, but instead, the widget itself masks the account number, see Select retail account.

  • Allow the bank customer to filter the arrangements that are presented, without additional server requests.

  • Allow the bank customer to select more than one arrangement.

View retail account details

The View retail account details widget of the Products capability displays detailed information for a selected product.

The actual set of information shown depends on the type of the selected product. Information available to this widget includes the following details:

  • General: Account type, IBAN, and available balance.

  • Interest-related: Interest rate, accrued interest, and minimum required balance.

  • Maturity: Monthly installment amount, principal amount, interest rate, and term.

  • Overdraft: Overdraft limit, and expiry date.

  • Interest and Redemption Settlement Account: Details of the account for receiving interest and for settling.

  • Associated Debit Cards: Masked card number and expiry date.

  • Account Opening Date and Last Updated Date: Date the account was opened and the last time account details were updated.

  • Status of the account: Active, Inactive, or Closed.

This widget interfaces with the Products DBS capability using the Products client API..

Features

  • Display the details of a selected product.

  • Unmask the account number. Click on the eye icon and the widget requests the unmasked account number from the server and displays it. This feature is only available if backend account masking is enabled.

The following features are configurable in Experience Manager:

  • Enable or turn off backend account masking.

    • If enabled, the widget requests account masking from the backend service, and if the backend service supports this feature, the backend service sends a masked account number to the widget. DBS supports this feature, see this page for more information.

    • If turned off, the widget does not request account masking from the backend service, but instead, the widget itself masks the account number, see View retail account details.

View retail account list

A product is a financial service offered by a bank or other financial institution to their customers. Backbase Platform classifies products by Product Kind. Product kinds are predefined categories that cannot be altered. These are the product kinds that the Backbase Platform supports:

  • Current Account – is a transactional, non-interest bearing account wherein a deposit is placed with the Bank for an unspecified period of time and the depositor can withdraw or transfer the funds whenever required through different means.

  • Savings Account – is an interest-bearing deposit account held at a bank or another financial institution that provides a modest interest rate. A savings account is the most basic type of account at a bank or credit union, allowing to deposit money and withdraw funds as needed.

  • Term Deposit – is a fixed-term deposit held at a financial institution. They are generally short-term deposits with maturities ranging anywhere from a month to a few years. When a term deposit is purchased, the money can only be withdrawn after the term has ended or by giving a predetermined number of days notice.

  • Debit Cards - is a payment card that is directly connected to the consumer’s checking account. During the transaction the amount of a transaction is withdrawn from the available balance in the cardholder’s account.

  • Credit Cards – is a payment card issued by a financial company which enables the cardholder to borrow funds which may further be used as payment for goods and services.

  • Loan – is a debt provided by a financial organization to another entity at an interest rate, specified by the attributes of the loan such as: Principal amount of money borrowed, the interest rate the lender is charging, and date of repayment. A loan may be for a specific, one-time amount or can be available as an open-ended line of credit up to a specified limit or ceiling amount.

  • Investment Account – is a type of financial account that contains a deposit of funds and/or securities that is held at a financial institution. The typical objectives of an Investment Account are to achieve long term growth, income or capital preservation from the deposited asset portfolio.

The widget displays the status of the account along with an icon. The possible statuses are:

  • Active

  • Inactive

  • Closed

Banks can also integrate their existing systems with the Products DBS capability in Backbase to catalog products, store arrangement details, and provide redundancy, that is, enrich and expand their existing systems with myriad of new features. The widgets in the widget collections related to the Products capability are made to work with DBS, which in turn can integrate with the existing systems at the bank and thus provide richer banking experience for the user.

This widget interfaces with the Products DBS capability using the Products client API.

Features

  • Display aggregated balances at the product kind level.

  • Display a total aggregated balance.

    • If turned off, the widget does not request account masking from the backend service, but instead, the widget itself masks the account number, see View retail account list.

Manage retail account settings

This widget enables bank customers to view and manage their accounts. Customers can set an alias for an account, and change its visibility.

If an alias, that is, a custom name is set for the account, it is displayed instead of the default account name. To set a custom name, the bank customer clicks the pencil icon next to the account name. To save the changes, users click the green ✔. To cancel, they click the red ✗.

To hide or show an account in the View retail account list widget, the bank customer clicks the slide control next to the account in Manage retail account settings.

The widget displays the status of the account along with an icon. The possible statuses are:

  • Active

  • Inactive

  • Closed

The widget has pagination in the web view and a Load more button in the mobile view.

This widget interfaces with the Products DBS capability using the Products client API.

Features

This widget offers the following features to bank customers:

  • View and manage accounts.

  • View the status of the account.

  • Set an alias for an account.

The following features are configurable in Experience Manager:

  • Enable or disable backend account masking.

    • If enabled, the widget requests account masking from the backend service, and if the backend service supports this feature, the backend service sends a masked account number to the widget. DBS supports this feature, see this page for more information.

    • If disabled, the widget does not request account masking from the backend service, but instead, the widget itself masks the account number, see Manage retail account settings.
