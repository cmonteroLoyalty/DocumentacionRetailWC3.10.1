Bill Pay v3.2.1

The highlights for this release are:

  • Web widgets

    The existing bill pay functionality was extended with new flows. Rush payment was added to extend the existing simple payment scenario. The Upcoming Payment widget was added to Retail Banking Widget Collection to represent future payments.

  • Native mobile widgets

    New functionality around electronic payees was added to the Bill Pay widget and client.

New features

The new features in this release are:

  • Web widgets

    • Rush payment flow.

    • Upcoming Payments widget.

  • Native mobile widgets

    • Manage bill payments: Electronic Payee

      • Payee summary list now includes the ability to add an electronic payee.

      • There is a new view to search for an electronic payee.

      • There is a new view to add account details to an electronic payee.

  • Native mobile clients

    • Bill Pay client

      • Bill pay now includes search payee functionality.

Bug fixes

The following issues have been fixed in this release:

  • Native mobile widgets

    • Bill Pay widget

      • Android: fix: bug in Mode.getEnum for EDIT_ADDRESS.

    • Bill Pay Form widget

      • Android: fix: handle the crash when account is not configured correctly.

  • Native mobile clients

    • Bill Pay client

      • Android: fix: throw from parsing methods instead of ambiguously returning null.

Deprecations

The following features are deprecated from this release:

  • Native mobile widgets

    • Bill Pay widget

      • Android: The Mode enum is deprecated as it is only necessary for internal usage in BillPayWidget. In the future its visibility will be lowered to package-private.

      • iOS: BillPayWidgetView and its subviews are deprecated. Copy and paste the source code to your own project.

Support for these features ceases in Retail Banking Widget Collection v3.4.0.

Breaking changes

The following breaking changes have been introduced in this release:

  • Native mobile widgets

    • Bill Pay widget: Default views are removed from Widget and moved to Sample Application, and following classes are renamed:

      • Android:

        • DefaultBillPayWidgetView has been renamed to BillPayWidgetView

        • DefaultBillPayWidgetPendingPaymentsView has been renamed to PendingPaymentsView

        • DefaultBillPayWidgetPaymentDetailsView has been renamed to PaymentDetailsView

        • DefaultBillPayWidgetPaymentEditView has been renamed to PaymentEditView

          Helper and util classes that existed to help the default views were moved in the same manner to the Sample Application as well.

      • iOS:

        • DefaultBillPayWidgetView has been renamed to BillPayWidgetView

        • DefaultBillPayWidgetPendingPaymentsView has been renamed to BillPayWidgetPendingPaymentsView

        • DefaultBillPayWidgetPaymentDetailsView has been renamed to BillPayWidgetPaymentDetailsView

        • DefaultBillPayWidgetPaymentEditView has been renamed to BillPayWidgetPaymentEditView

        • DefaultBillPayWidgetViewDataSource has been renamed to BillPayWidgetViewDataSource

        • DefaultBillPayWidgetViewPaymentsDataSource has been renamed to BillPayWidgetViewPaymentsDataSource

        • DefaultBillPayPaymentCell has been renamed to BillPayPaymentCell

          Helper and util classes that existed to help the default views were moved in the same manner to the Sample Application as well.

Known issues

  • DLRN-170 - Discard changes flow.

  • DLRN-118 - Discard changes flow.

Products v3.2.1

The highlights for this release are:

  • Can view custom product kinds

  • Can mask account numbers

New feature

The new feature in this release is:

  • Added the ability to view custom product kinds.

  • Added the ability to mask account numbers so that only the final four digits are displayed. The remainder of the account number is replaced with bullet points (•).

Deprecations

The following features are deprecated in this release:

  • Native mobile clients

    • Product Summary client:

      All properties that uses DebitCardItem will be affected by the following change:

      • iOS: DebitCardItem class is deprecated and replaced with BBDebitCardItem.

      • Android: DebitCardItem class is deprecated and replaced with DebitCardItem from common utils.

    • Accounts client:

      • ArrangementItem: legalEntitiesIds is deprecated, use legalEntityIds instead.

      • ArrangementItemBase#alias is deprecated and replaced by ArrangementItemBase#bankAlias. Currently both properties still point to the "alias" backend field, but bankAlias will point to the "bankAlias" backend field as soon as it is available. alias will eventually be completely removed.

      • Android:

        • All of the setters in ArrangmentItem and ArrangementItemBase are deprecated.

        • updateUserPreferences method is deprecated, use updateUserPreferences method instead, which has UserPreferencesItem from Common Utils.

      • iOS:

        • updateUserPreferencesWithUserPreferencesItem method is deprecated, use updateUserPreferencesWithUserPreferences method instead.

Support for these features ceases in Retail Banking Widget Collection v3.4.0.

Bug fixes

The following issues were fixed in this release:

  • Native mobile widgets

    • Product Summary widget

      • iOS: fix: crash on account details.

      • Android: fix: crashes when running with realServerSettings variant.

  • Native mobile clients

    • Product Summary client

      • Android: fix: make ProductKind extend Additions.

Known issues

The known issue in this release is:

  • PRSUM-4576 - When unmasking a field for a selected account, all masked fields for all accounts are unmasked.

Transactions v3.2.1

New features

The new features in this release are:

  • Web widgets

    • Sorting functionality is now available in the Transactions List widget.

      Projects can now use this functionality to offer the end-user different viewing options.

    • A new preference has been introduced in Experience Manager to configure the default sorting field and direction, and to indicate whether the widget reflects sorting information that is returned in the backend response.
