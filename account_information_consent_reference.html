Selector name

This widget has the following selector name: bb-consent-request-widget

Required imports

The following imports are required to set up and customize this widget:

  • Widget module import

    Import the following modules into the module of your wrapper widget:

    import { ConsentRequestWidgetModule } from '@backbase/consent-request-widget-ang';

  • Data module imports

    Import the following data modules into the module of your wrapper widget:

    import { ConsentDataHttpModule } from '@backbase/data-ang/consent';

Extension slots

This widget has the following extension slots:

  1. bbConsentRequestWidgetCustomizable

  2. bbConsentRequestTppInformationCustomizable

  3. bbConsentRequestAccountSelectorCustomizable

  4. bbConsentRequestPermissionsInformationCustomizable

bbConsentRequestWidgetCustomizable

You configure this extension slot using:

<ng-container
  bbConsentRequestWidgetCustomizable
  [bbHostRef]="hostRef"
  [bbTemplateContext]="{
    consentRequest: consentRequest | async,
    multiAccountSelection: multiAccountSelection$ | async
  }">
</ng-container>

<ng-template
  bbConsentRequestWidgetCustomizable
  let-hostRef
  let-consentRequest="context.consentRequest"
  let-multiAccountSelection="context.multiAccountSelection"
>
  <ng-container *ngIf="hostRef.consentRequestId | async; else errorTemplate">
    <ng-container *ngIf="!(hostRef.loading | async) as loading; else loadingTemplate">
      <ng-container *ngIf="!(hostRef.error | async); else errorTemplate">
        <div class="bb-consent-request" *ngIf="consentRequest">
          <div class="bb-heading-2 bb-block--lg"
            data-role="request-access-label"
            i18n="Widget header|Third party request access message@@consent-request.label.header">
            Third party requested access to your accounts
          </div>

          <ng-container *ngIf="!(hostRef.saving | async); else savingLoadingTemplate">
            <bb-alert-ui
              *ngIf="hostRef.savingError | async"
              title="Unknown error"
              i18n-title="Unknown Error|Unknown Error Label@@consent-request.label.error.unknown">
              <span data-role="allow-consent-request-error"
                i18n="Try again|Try again error message@@consent-request.label.error.tryAgain">
                Try to make action again or contact your system administrator.
              </span>
            </bb-alert-ui>

            <div class="bb-block--lg">
              <bb-consent-request-tpp-information
                [tpp]="consentRequest.tpp"
              ></bb-consent-request-tpp-information>
            </div>

            <div class="bb-block--lg">
              <bb-consent-request-account-selector
                [multiAccountSelection]="multiAccountSelection"
                [showError]="hostRef.showAccountError"
                [accounts]="consentRequest.accounts"
                [(selectedAccount)]="hostRef.selectedAccount"
              ></bb-consent-request-account-selector>
            </div>

            <div class="bb-block--lg">
              <bb-consent-request-permissions-information
                [permissions]="consentRequest.permissions"
              ></bb-consent-request-permissions-information>
            </div>

            <div class="bb-block--lg">
              <div class="bb-block--sm bb-text-bold"
                data-role="expiration-date-label"
                i18n="Expiration date|Consent expiration date@@consent-request.label.expiryDate">
                Expiration date
              </div>

              <div>{{consentRequest.expiryDate | bbDate:'bbShort'}}</div>
            </div>
          </ng-container>

          <div class="bb-button-bar bb-button-bar--reverse">
            <button
              bbButton
              [disabled]="hostRef.saving | async"
              (click)="hostRef.allowConsentRequest()"
              i18n="Allow button|Allow consent request@@consent-request.label.allow"
              color="primary"
              class="bb-button-bar__button"
              data-role="allow-button">
              Allow
            </button>
            <button
              bbButton
              [disabled]="hostRef.saving | async"
              (click)="hostRef.rejectConsentRequest()"
              i18n="Reject button|Reject consent request@@consent-request.label.reject"
              color="secondary"
              class="bb-button-bar__button"
              data-role="reject-button">
              Reject
            </button>
          </div>

          <ng-template #savingLoadingTemplate>
            <div class="bb-state-container">
              <bb-loading-indicator-ui
                text="Processing the request..."
                i18n-text="Processing|Processing the allow/reject request@@consent-request.label.processing"
              ></bb-loading-indicator-ui>
            </div>
          </ng-template>
        </div>
      </ng-container>
    </ng-container>

    <ng-template #loadingTemplate>
      <div class="bb-state-container">
        <bb-loading-indicator-ui
        ></bb-loading-indicator-ui>
      </div>
    </ng-template>
  </ng-container>

  <ng-template #errorTemplate>
    <div class="bb-state-container">
      <bb-consent-request-error
        [consentRequestId]="hostRef.consentRequestId | async"
        [errorKind]="(hostRef.error | async)?.kind"
      ></bb-consent-request-error>
    </div>
  </ng-template>

  <ng-template #notificationError>
    <span i18n="Operation Failed|Error notification when failing to save request consent@@consent-request.label.notification.error">
      Operation failed
    </span>
  </ng-template>
</ng-template>

This extension slot consumes the following object properties from its custom context:

Object property Description

readonly hostRef: ConsentRequestContainerComponent

Component context.

Consent request body.

Multi-account selection usage flag. If true, multi-selection account selector is enabled. If false a single selection account selector is enabled. Retrieving value from CXP manager.

bbConsentRequestAccountSelectorCustomizable

You configure this extension slot using:

<ng-container
  bbConsentRequestAccountSelectorCustomizable
  [bbHostRef]="hostRef"
  [bbTemplateContext]="accounts">
</ng-container>

<ng-template
  bbConsentRequestAccountSelectorCustomizable
  let-hostRef
  let-accounts="context">
  <div class="bb-block bb-block--sm bb-text-bold"
       *ngIf="!hostRef.selectedAccount || (hostRef.selectedAccount && hostRef.selectedAccount.length <= 1)"
    i18n="Account Exposed|Account Exposed label@@consent-request.label.accountExposed"
    data-role="account-exposed-label">
    Account Exposed
  </div>
  <div class="bb-block bb-block--sm bb-text-bold"
       *ngIf="hostRef.selectedAccount && hostRef.selectedAccount.length > 1"
    i18n="Accounts Exposed|Account Exposed label@@consent-request.label.accountsExposed"
    data-role="accounts-exposed-label">
    Accounts Exposed
  </div>

  <div [ngClass]="{ 'border border-danger rounded': hostRef.showError }">
    <bb-account-selector-ui
      data-role="account-selector"
      [items]="accounts"
      [disableScrollEnd]="false"
      [multiple]="hostRef.multiAccountSelection"
      [closeOnSelect]="!hostRef.multiAccountSelection"
      [filterItems]="false"
      [selectedItems]="hostRef.multiAccountSelection ? hostRef.selectedAccount : (hostRef.selectedAccount ? hostRef.selectedAccount[0] : undefined)"
      [dropdownPosition]="'bottom'"
      (change)="hostRef.onSelectAccount($event)"
      placeholder="Select Accounts"
      i18n-placeholder="Select Accounts placeholder@@consent-request.account-selector.placeholder"
    >
      <ng-template bbCustomSingleSelectedItemLabel let-item="item">
        <bb-product-item-basic-account-ui
          [active]="true"
          [amount]="item.balance"
          [currency]="item.currency"
          [productNumber]="item.IBAN || item.BBAN || item.number"
          [title]="item.name"
        >
          <ng-template bbCustomProductItemBasicAccountDetailsFormatter let-amount="amount" let-currency="currency">
            <bb-amount-ui
              class="font-weight-bold"
              [amount]="amount"
              [currency]="currency"
              [mapCurrency]="false"
            ></bb-amount-ui>
          </ng-template>

        </bb-product-item-basic-account-ui>
      </ng-template>
      <ng-template bbCustomOptionItem let-item$="item$" let-item="item">

            <bb-product-item-basic-account-ui
              [active]="item$.selected"
              [productNumber]="item.IBAN || item.BBAN || item.number"
              [amount]="item.balance"
              [currency]="item.currency"
              [title]="item.name"
            >
              <ng-template bbCustomProductItemBasicAccountDetailsFormatter let-amount="amount" let-currency="currency">
                <bb-amount-ui
                  class="bb-highlight bb-product-item__amount"
                  [amount]="amount"
                  [currency]="currency"
                  [bbHighlight]="true"
                ></bb-amount-ui>
              </ng-template>
            </bb-product-item-basic-account-ui>
      </ng-template>
    </bb-account-selector-ui>
  </div>
  <div
    *ngIf="hostRef.showError"
    class="text-danger mt-2"
    data-role="no-accounts-error-message"
    i18n="Select an account|No account selected label@@consent-request.label.noAccountSelected">
    Select an account.
  </div>
</ng-template>

Implementation example

The following examples show the implementation of this widget:

  • Wrapper widget module

    For example: .NewExtensionSlot.module.ts

import { NgModule } from '@angular/core';
import { CommonModule } from '@angular/common';
import { BackbaseCoreModule } from '@backbase/foundation-ang/core';
import { ConsentRequestWidgetModule } from '@backbase/consent-request-widget-ang';
import { CustomWidgetComponent } from './custom-widget.component';

@NgModule({
  declarations: [CustomWidgetComponent],
  imports: [
    CommonModule,
    BackbaseCoreModule.withConfig({
      classMap: { CustomWidgetComponent },
    }),
    ConsentRequestWidgetModule,
  ],
  exports: [
    ConsentRequestWidgetModule,
    CustomWidgetComponent,
  ],
})
export class CustomWidgetModule { }

  • Wrapper widget component

    For example: .NewExtensionSlot.component.ts

import { Component, EventEmitter, Output } from '@angular/core';

@Component({
  selector: 'bb-custom-widget',
  templateUrl: './custom-widget.component.html',
  styles: []
})
export class CustomWidgetComponent {
  @Output() allowConsentRequestRedirectUrl = new EventEmitter<string>();
  @Output() rejectConsentRequestRedirectUrl = new EventEmitter<string>();
}

  • Custom template For example: .NewExtensionSlot.component.html

<bb-consent-request-widget
  (allowConsentRequestRedirectUrl)="allowConsentRequestRedirectUrl.emit($event)"
  (rejectConsentRequestRedirectUrl)="rejectConsentRequestRedirectUrl.emit($event)"
></bb-consent-request-widget>
