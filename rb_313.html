Retail Banking Widget Collection v3.1.3

The highlights for this release are:

  • Web widgets

    • PSD2: approve/reject AISP consent (new)

    • PSD2: list AISP Consents (new)

    • PSD2: revoke AISP Consent (new)

    • PSD2: approver/reject PISP consent (new)

    • Message Centre: create Message (new)

    • Card management: configure overview based on the card type (debit/credit)

    • Product Summary: several accessibility improvements

  • Native mobile clients

    • Payment Order: US and custom payment types are now supported

    • Bill Pay: easily manage electronic payees

Authentication v3.1.3

This release note lists the changes for the Authentication capability in this release.

Deprecations

The following features are deprecated from this release. Support for these features ceases in v3.2.0.

  • Native mobile widgets

    • Passcode Widget

      iOS

      • The usage of the confirmEvent and setupEvent events changes: they will not require a proxy implementation when the planned changes are made. Both events now can be registered through the offline model or the views, they are already published through the widget.

Bill Pay v3.1.3

This release note lists the changes for the Bill Pay capability in this release.

New features

The new features in this release are:

  • Native mobile clients

    • Using the Bill pay you can now add, remove, delete and update electronic payees.

Deprecations

The following features are deprecated from this release. Support for these features ceases in v3.2.0.

  • Native mobile widgets

    • Bill Pay Form widget

      In the BillPayFormView class, the updateViewWithPayee method is deprecated. Instead use the updateViewWithPayeeSummary method.

      • The GenericListener and methods that accept it will be removed, and replaced with method-specific listeners. The versions of each method with specific listeners are already available.

      Cross-platform

      • The createConversationSelectedAction property is deprecated, instead use the createConversationAction property.

      • The replyMessageSelectedAction property is deprecated, instead use the replyMessageAction property.

      • The conversationSendingDidFinish delegate method is deprecated, instead use the didFinishSendingDraft delegate method.

Known issues

The known issues in this release are:

    • DBSK-2705 - text editor formatting tags are counted as symbols. Workaround: increase character limit or disable rich text editor tools

Payments v3.1.3

This release note lists the changes for the Payments capability in this release.

Deprecations

The following features are deprecated from this release. Support for these features ceases in v3.2.0.

  • Web widgets

    • The 'Initiate SEPA Payment' widget has been replaced by a more generic and configurable 'Initiate Payment' widget. By default, configured for SEPA Credit transfers.

  • Native mobile widgets

    • Payments Widget

      This widget is deprecated, instead use the PaymentOrderWidget.

Known issues

On the 'Initiate Paymentâ€™ widget, there are the following known accessibility related issues:

  • When a user inputs an invalid IBAN, the reader will announce it only as long as the user is editing the IBAN. If the user navigates away from the IBAN, the reader will not pick up warning message for invalid IBAN.

  • When a user searching for a beneficiary, the reader reads an extra search results taking into account the debit account as well if it matches the search pattern.

  • On Firefox, when a user is using using a tabbed navigation, he cannot tab out of the amount field to access the rest of the fields.

Places v3.1.3

This release note lists the changes for the Places capability in this release.

Known issues

  • Native mobile

    • MAINT-8626: App Store rejection due to use of non-public API in Places widget for iOS.

      The Places widget for iOS version has a blocker for App Store release. For details, visit the Community.

      This issue was fixed in Widget Collection Retail Banking v3.2.1. Upgrade to this version. When upgrading to v3.2.1, consider the following:

      Native mobile widgets have a new technical version number as of Widget Collection Retail Banking v3.2.0. Until v3.2.0, the widget collection has followed the version format <release>.<major>.<minor>.<patch>. As of the v3.2.0 major release, the <release> digit has been removed from artifacts. This means that, from that release on, the native mobile widgets are versioned as v2.1.0 in the Backbase repository instead of 3.2.1(.0). This change was introduced because customers with patched versions of the collection, for example 3.2.1.1, were unable to publish apps to the App store. This is due to the App Store Connect rules around apps with 4-digit version numbers. The <release> number is now part of the repository URL, which is the version number developers add to their local CocoaPods and Gradle installations. This affects both for Android and iOS.

      To use the new major version of Widget Collection Retail Banking, update the Gradle and Cocoapods dependencies to reflect the repository name change. It is important to understand that the actual product version is still 3.2.1(.0).

      For example:

      pod 'PlacesWidget', '2.1.0'
pod 'TransactionsWidget', '2.1.0'

Products v3.1.3

This release note lists the changes for the Products capability in in this release.

New features

The new features in this release are:

  • Web widgets

    • Keyboard AA accessibility: navigation using the tab and space keys is implemented.

    • Screen reader AA accessibility improvements.

Deprecations

The following features are deprecated from this release. Support for these features ceases in v3.2.0.

  • Native mobile widgets

    • Product Summary Widget

      Warning
      		 The following deprecations apply only when the Accounts Widget is available.

      Android

      • The ProductSummaryContract#selectedAccount(Product) method is deprecated.

      • The ProductSummaryView#updateViewWithProduct(Product) method is deprecated.

      • The EVENT_ACCOUNT_SELECTED event is deprecated

      • The EVENT_ACCOUNT_RECEIVED event is deprecated.

      iOS

      • The updateViewWithProduct method is deprecated.

      • The accountReceivedEvent event with bb.action.product-summary.account-received is deprecated, instead use the accountSelectedEvent event.

      • The productReceivedEvent event with bb.action.product-summary.product-received is deprecated, instead use the productSelectedEvent event.

      • The didSelectProduct method is deprecated, instead use the productSelected method.

    • Product Summary Client

      Android

      • The productKindName field in the Product class is deprecated.

      • The type of the bookedBalance, availableBalance, creditLimit, currentInvestmentValue, and interestSettlementsAccount fields will be changed from string to BigDecimal in the subclasses of Product.

      • The type of the AggregatedBalance field will be changed from string to BigDecimal.

      • The type of the principalAmount will be changed from string to number in the subclasses of Product.

      iOS

      • In the Product class, the type of the bookedBalance, availableBalance, creditLimit, currentInvestmentValue and interestSettlementAccount properties will be changed to decimal.

      • In the Product class, the defaultAmount and productKindName properties are deprecated and will be removed.

      • In the AggregatedBalance class, the type of the aggregatedBalance property will be changed to decimal.

Remote Deposit Capture v3.1.3

This release note lists the changes for the Remote Deposit Capture capability in this release.

Deprecations

The following features are deprecated from this release. Support for these features ceases in v3.2.0.

  • Native mobile widgets

    • RDC Client

      Android

      • The getRDCDailyLimit method with the rdcDepositDailyLimitListener is deprecated, instead use the getRDCDailyLimit method with the rdcDailyLimitListener.

      • The RDCDailyLimit class is deprecated, instead use the Amount class from CommonUtils.

      • The AggregatedBalance class is deprecated, instead use the Amount class from CommonUtils.

      • For the RDCDeposit class, the amount and currency properties are deprecated, instead use the depositAmount property which has the type Amount.

      • For the RDCFormData class, the amount and currency properties are deprecated, instead use the formDataAmount property which has the type Amount.

      • For the RDCProduct class, the balance, availableBalance and currency properties are deprecated, instead use the productBalance and the productAvailableBalance properties which have the type Amount.

      • For the RDCProductKind class, the type of the aggregatedBalance property will change to Amount. As an alternative, use the Amount conversion methods in the AggregatedBalance class.

      iOS

      • The loadRDCDailyLimitWithHandler method is deprecated, instead use the loadRDCDailyLimit method.

      • The AggregatedBalance class is deprecated, instead use BBCurrency from CommonUtils.

      • For the RDCDeposit class, the amount and currency properties are deprecated, instead use the depositAmount property which has the type BBCurrency from CommonUtils.

      • For the RDCFormData class, the amount and currency properties are deprecated, instead use the formDataAmount property which has the type BBCurrency from CommonUtils.

      • For the RDCProduct class, the balance, availableBalance and currency properties are deprecated, instead use the productBalance and the productAvailableBalance properties which have the type BBCurrency from CommonUtils.

      • For the RDCProductKind class, the type of the aggregatedBalance property will change to BBCurrency. As an alternative, use the BBCurrency conversion methods in the AggregatedBalance class.

Transactions v3.1.3

This release note lists the changes for the Transactions capability in this release.

Deprecations

The following features are deprecated from this release. Support for these features ceases in v3.2.0.

  • Native mobile widgets

    • Transactions Widget

      Android

      • The EVENT_TRANSACTION_RECEIVED event will be removed. Use #onTransactionSelected(TransactionItem) along with model-defined navigation instead.

      iOS

      • The transactionSelectedEvent event is deprecated, instead use the transactionSelectedAction property.

      • The transactionReceivedEvent pub/sub event is deprecated.

    • Transactions Client

      Android

      • The getTurnOvers method with individual parameters is deprecated. Instead use getTurnOvers with TurnOversRequest object.

      • The getCategoryTotals method with individual parameters is deprecated. Instead use getCategoryTotals with CategoryTotalsRequest object.

      • The Amount class is deprecated. All usages of this type will be converted to use the Amount class from common-utils.

      • For the TransactionItem class, the amount, currency, instructedAmount and instructedCurrency fields are deprecated and replaced with the transactionAmountCurrency and instructedAmountCurrency fields. Both new fields are of the type Amount.

      iOS

      • The retrieveTurnoversForArrangementIds method is deprecated. Instead use the retrieveTurnoversWithTurnoversRequest method.

      • The retrieveCategoryTotalsForCreditDebitIndicator method is deprecated. Instead use the retrieveCategoryTotalsWithCategoryTotalsRequest method.

      • The Amount class is deprecated. All usages of this type will be converted to use the BBCurrency class from common-utils.

      • For the TransactionItem class, the amount, currency, instructedAmount and instructedCurrency fields are deprecated and replaced with the transactionAmountCurrency and instructedAmountCurrency fields. Both new fields are of the type BBCurrency.
