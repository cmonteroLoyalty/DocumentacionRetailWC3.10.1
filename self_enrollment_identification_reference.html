Provide information for self-enrollment

Preferences

The widget has the following preferences that managers or administrators can edit in Experience Manager:

Title and description Name Admin[a] Default value

Locked out info title

Sets the title of a message that appears in case the user gets locked out from the identification process.

lockedOutTitle

X

Contact Us

Locked out info message

Sets the content of a message that appears in case the user gets locked out from the identification process.

lockedOutMessage

X

Call Us 404-555-0123 working days between 9:00 am and 5:30 pm. Find the branch nearest to you.

Cancel navigation

Sets a navigation path when the user requests to cancel the enrollment process.

output.cancel

X

navigation:

Complete navigation

Sets a navigation path when the user completes the enrollment process and submits the identification form.

output.complete

X

navigation:

[a] Preferences that have this column checked, are only editable for administrators.

Access Control

The bank customer does not require any permissions to access this widget’s functionality.

Extension slots

This widget has the following extension slots:

  1. bbSelfEnrollmentIdentificationCustomizable

  2. bbSelfEnrollmentIdentificationFormFieldsCustomizable

  3. bbSelfEnrollmentIdentificationErrorCustomizable

  4. bbSelfEnrollmentIdentificationWarningCustomizable

bbSelfEnrollmentIdentificationCustomizable

You configure this extension slot using:

<ng-template bbSelfEnrollmentIdentificationCustomizable let-hostRef>
  <div class="card card-lg bb-block bb-block--lg">
    <div class="card-body">
      <div class="bb-block bb-block--lg">
        <bb-header-ui
          heading="Personal Information"
          headingType="h2"
          i18n-heading="
              Personal Information|Header for self enrollment identification
            form@@self.enrollment.identification.header
          "
        ></bb-header-ui>
      </div>

      <ng-container bbSelfEnrollmentIdentificationErrorCustomizable [bbHostRef]="hostRef"></ng-container>

      <ng-container bbSelfEnrollmentIdentificationWarningCustomizable [bbHostRef]="hostRef"></ng-container>

      <div class="bb-stack bb-stack--wrap no-gutters">
        <div class="bb-stack__item bb-stack__item--align-top col-12 col-md-6">
          <div class="bb-block bb-block--sm bb-stack bb-stack--wrap">
            <bb-icon-ui
              class="bb-stack__item bb-stack__item--align-top"
              name="verified-user"
              color="success"
              size="xxl"
            ></bb-icon-ui>
            <div class="bb-stack__break bb-stack__break--spacing-sm bb-stack__break--md-up"></div>
            <p
              class="bb-stack__item bb-stack__item--fill bb-text-support"
              i18n="
                  Identification information usage|Short description on how identification information is being
                used@@self.enrollment.identification.description
              "
            >
              This information will be only used to validate your account and won't be shared with any external data
              services.
            </p>
          </div>
        </div>

        <div class="bb-stack__break bb-stack__break--sm-down"></div>
        <div class="bb-stack__item bb-stack__item--fill">
          <form
            [formGroup]="hostRef.identificationForm"
            (keyup.enter)="hostRef.onSubmit()"
            data-role="identification-form"
          >
            <ng-container
              bbSelfEnrollmentIdentificationFormFieldsCustomizable
              [bbHostRef]="hostRef"
              [bbTemplateContext]="hostRef.identificationForm"
            ></ng-container>
          </form>
        </div>
      </div>
    </div>
  </div>

  <div class="bb-button-bar">
    <button
      bbButton
      class="bb-button-bar__button"
      data-role="continue-button"
      color="primary"
      i18n="Continue|Self enrollment identification submission button label@@self.enrollment.identification.continue"
      (click)="!hostRef.isIdentifying && !hostRef.isLockedOut && hostRef.onSubmit()"
    >
      Continue
    </button>
    <button
      bbButton
      class="bb-button-bar__button"
      color="link"
      i18n="Cancel|Cancel self enrollment identification button label@@self.enrollment.identification.cancel"
      [disabled]="hostRef.isLockedOut"
      (click)="hostRef.onCancel()"
    >
      Cancel
    </button>
  </div>
</ng-template>

bbSelfEnrollmentIdentificationErrorCustomizable

You configure this extension slot using:

<ng-template bbSelfEnrollmentIdentificationErrorCustomizable let-hostRef>
  <bb-alert-ui
    data-role="could-not-identify-error-message"
    *ngIf="hostRef.isNotIdentified"
    modifier="error"
    title="Unfortunately we could not verify your information"
    i18n-title="
        We could not identify you|Title of an error message that appears in case user's identification has
      failed@@self.enrollment.identification.error.invalid.title
    "
    i18n="
        The data does not match your profile data|Explanation on why the identification process has
      failed@@self.enrollment.identification.error.invalid.text
    "
  >
    The data you entered does not match your profile data. Please try again.
  </bb-alert-ui>
  <bb-alert-ui
    data-role="locked-out-error-message"
    *ngIf="hostRef.isLockedOut"
    modifier="error"
    title="Unfortunately you are locked out"
    i18n-title="
        You are locked out|Title of an error message that appears in case users gets locked out during
      identification@@self.enrollment.identification.error.lock.title
    "
    i18n="
        You have exceeded allowed number of attempts|Explanation on why is the user being locked out from
      identification process@@self.enrollment.identification.error.lock.text
    "
  >
    You have exceeded the allowed number of attempts. Please contact us for further assistance.
  </bb-alert-ui>
  <bb-alert-ui
    data-role="identify-system-error-message"
    *ngIf="hostRef.isSystemErrorOccurred"
    modifier="error"
    title="Unexpected error"
    i18n-title="
        Unexpected error|Title of an error message that appears in case when system is unable to process
      identification data@@self.enrollment.identification.error.system.title
    "
    i18n="
        Please try again later|Content of an error message that appears in case when system is unable to process
      identification data@@self.enrollment.identification.error.system.text
    "
  >
    Something is gone wrong with our server. Please try again later.
  </bb-alert-ui>
</ng-template>

bbSelfEnrollmentIdentificationWarningCustomizable

You configure this extension slot using:

<ng-template bbSelfEnrollmentIdentificationWarningCustomizable let-hostRef>
  <bb-alert-ui
    data-role="locked-out-info-message"
    *ngIf="hostRef.isLockedOut"
    modifier="info"
    [title]="hostRef.lockedOutTitle"
  >
    {{ hostRef.lockedOutMessage }}
  </bb-alert-ui>
</ng-template>
