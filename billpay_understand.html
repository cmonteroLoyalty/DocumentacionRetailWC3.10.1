Understand Bill Pay

Note
 This section describes an add-on. You may use this add-on only if you have purchased an appropriate license for this add-on based on a signed Product Order Form.

Bill Pay is a payment system in the US that allows consumers to pay a person or business online electronically or by check. Consumers in the US pay many individual bills on a monthly basis, including gas, electricity, cable, internet, telephone, credit cards, and more. Bill Pay allows them to pay these bills online directly on the provider’s website, or through the consumer’s bank or Credit Union, instead of having to write and post a check themselves. Some providers support eBill functionality, which means that the consumer can receive an electronic version of their bill.

Bill pay in its own right is not strictly an online banking concept, but banks are able to integrate with the bill pay ecosystem. Using bill pay through online banking instead of through provider websites is advantageous though, because:

  • It’s easier and faster for consumers to organize and pay all of their bills from one central location (their online banking portal), as opposed to having to go to every provider’s website separately.

  • It makes the consumer less likely to suffer late fees, because they can see all of their bills in one place and set up recurring payments.

  • It saves the user time and money, as they don’t need to write a check; buy a stamp; or go to a post office to pay any of their bills.

  • It allows the user to a history of previous payments to their providers.

Backbase’s Bill Pay capability is made for the US market. Bill Pay always works in conjunction with an external bill pay provider for the actual processing of the transaction. The Bill Pay widgets send the bill pay request to the backend, which is either Backbase’s own DBS or a third-party system. The bill pay request ends up with an external bill pay provider that processes the bill pay request and either transfers the money via an electronic ACH payment or by mailing a physical check.

Note
 In addition to Bill Pay, Backbase offers these other money transfer capabilities: Payments for users to make a direct payment, and Remote Deposit Capture that allows bank users to scan and deposit a check using their mobile phone camera.

Enrol a user in bill pay

Check if a user is enrolled, if not, sign up the user so they have permissions to use Bill Pay. The data for enrollment is currently provided in the integration service. The integration service is a project-specific, custom backend service that ties DBS Bill Pay or widgets to the external bill pay provider. The enrollment flow also allows for data migration of Bill Pay accounts in the enrollment process.

Pay a bill and manage payees

A payee is the company or an individual that has issued the bill and is the recipient of the money transfer. Users have the following bill pay options for each payee:

  • Make a one-time payment, including an eBill related payment.

  • Make a recurring payment.

Pay a bill and manage payees has the following features:

  • List payees

    For each payee the following data is available: name, amount, deliver by (date), pay from (account selector), available balance on the selected account, whether eBill is available, next recurring payment if applicable, next due eBill if applicable and buttons to add a memo, edit the payee, set up a recurring payment and to set up eBill for the payee.

  • View payee summary

    Displays a summary of details for a given payee with the following data available: payee name, payee type, payee account number, payee address, and payee phone number. Buttons are available to edit or delete the payee and to make a payment to the payee. Payee summary allows navigating between all payees. If no payees are returned, a button to add a new payee is provided.

  • Create a payee

    The list of payees shows a create button that triggers a create form. It is also possible to use the create a payee widget stand-alone (without the list of payees).

    The create form has two modes: search for a payee known to the external bill pay provider and enter a manual payee. Manual payee data include name, address, city, state, zip code, phone number, account number (optional) and nick name (optional). For a known payee, users can enter the account number (optional), nick name (optional) and the zip code, if that is required for that payee.

  • Edit a payee (only supported by web widgets)

    The list of payees has the option to edit a payee, which triggers an edit form.

    The edit form has two modes: it shows different fields for a payee known to the external bill pay provider and for a payee that was created manually. Manual payee data include name, address, city, state, zip code, phone number, account number (optional) and nick name (optional). For a known payee, users can enter the account number (optional), nick name (optional) and the zip code, if that is required for that payee.

  • Create one-time bill payments

    The list of payees allows to immediately create a new payment by setting amount, date, memo and account. In the web widget, multiple payments can be created at once. The user is allowed to select a date for the payment on or after the first allowable payment date.

    If rush payments are available this is indicated and the first possible rush date and rush fee are displayed. If the rush date is selected, the rush fee is added to the amount and the fee is indicated. When payments are created successfully, all fields are cleared.

    If one or more payments fail for any reason error messages are displayed.

  • Create or edit a recurring bill payment (only supported by web widgets)

    The list of payees has the option to set up a recurring payment, which triggers an edit form. It is also possible to use the recurring bill payment widget stand-alone (without the list of payees).

    Users select from the list of payees, select their account to pay from, set the amount and the first payment date, select the frequency and when the recurring payments will end. The frequency options are weekly, bi-weekly, semi-monthly, monthly, quarterly, semi-annually, or annually. The recurring payment either never ends, ends after a fixed number of occurrences, or ends on a particular date.

    Users can choose to receive a notification when the payment is pending, when the payment has been sent, and/or when the last payment is pending.

The Edit a payee and Create or edit a recurring bill payment features are only supported by the web widgets of Widget Collection Retail Banking.

Manage pending bill payments

The pay a bill and manage payees functionality support scheduling bill payments to be executed in the future, either as a one-time or as a recurring bill payment. The manage pending bill payments functionality lists future bill payments and allows users to manage them.

Manage pending bill payments has the following features:

  • List pending bill payments

    Lists bill payments with the states pending and in-process, and shows the following data for each bill payment: date, amount, payment type, payment details.

  • Edit a bill payment

    Users can edit only one-time bill payments that have the state pending.

  • Delete a bill payment

    Users can delete bill payments (one-time or recurring) that have the state pending.
