Web widgets

Note
 This section describes an add-on. You may use this add-on only if you have purchased an appropriate license for this add-on based on a signed Product Order Form.

The web widgets included in Bill Pay are:

  • View bill payees

  • View bill payee summary

  • View bill payments

  • Manage bill payee

  • Manage bill payments

  • Guard bill pay enrollment

  • Enroll bill pay automatically

  • Enroll Payveris eBills

View bill payees

In this widget, the bank customer views a list of payees.

By default, the bank customer clicks Pay or â–¼ to access the Bill Pay Widget, then chooses Make a payment or Set up a recurring payment.

This widget interfaces with the Billpay DBS capability using the Bill Pay client API.

Related topics

For further information, see:

  • Reference - View bill payees

View bill payee summary

In this widget, bank customers navigate between payees and see detailed information about each payee.

The information includes the following:

  • Payee name

  • Payee type (Electronic or Check)

  • Payee account number

  • Payee nickname

  • Payee address (only for Check payees)

  • Payee phone number (only for Check payees)

The customer can delete the payee in this widget, or click through to edit the payee or make a payment in other Bill Pay widgets.

This widget interfaces with the Bill Pay DBS capability using the Bill Pay client API.

Features

This widget provides the following features for the bank customer:

  • Navigate between payees.

  • View summary information about a payee.

  • Delete a payee.

  • Navigate to editing a payee in the Manage bill payee widget.

  • Navigate to making a payment to the payee in the Manage bill payments widget.

Developers set up the navigation between widgets in Experience Manager using widget preferences.

Manage bill payments

In this widget, bank customers make one-time or recurring payments to the selected payee.

The bank customer takes the following steps:

  1. Specify To and From accounts. Developers can configure the default currency used by the widget before the From account is selected in widget preferences. Once the account is selected, the widget uses the currency of that account.

  2. Provide payment details. If, according to the selected To account, the customer is making an eBill payment, then the widget displays a message about the eBill due date along with the following amount options:

    • The amount due on the latest billing cycle

    • The whole amount due

    • The minimum amount due

    • The custom amount that the customer enters manually

    For other To accounts, only the custom amount option is available.

    Developers can configure the Payment details title in widget preferences.

  3. (Optional) Provide payment description.

  4. Set payment schedule.

    For Frequency, a Once payment requires only a payment date. For payment frequencies higher than Once, the customer must set a start and end date, or a number of payment occurrences. A customer can optionally specify a Never end date.

    If the frequency is set to more than once, the widget displays notification options, where the customer can choose to get notified every time a scheduled payment is pending or sent, and/or when the last payment of the series is pending.

    When the customer sets a specific end date for the payment, the widget displays a message with the estimated number of occurrences. When the customer specifies the number of occurrences, the widget displays a message with the estimated end date.

Once all the payment details are provided, the customer can review the payment before submitting it. If the selected date falls on a weekend or any other day when the payment cannot be made, the widget displays a corresponding error.

Guard bill pay enrollment

This widget is designed to host the Deck Component containing the Bill Pay components. It checks the user’s enrolment status. If they are enrolled, they are allowed to view the pages, if not, they are re-routed to the Auto-Enrolment widget.

The user’s enrolment status is checked using: Adopt Bill Pay with the REST API

Features

This widget has the following features:

  • Checks current enrolment status.

  • Allows access to sub-pages only if the user is enrolled.

  • Can re-route the user to the auto-enrolment widget if they are not enrolled.

Related topics

For further information, see:

  • Reference - Guard bill pay enrollment

Enroll bill pay automatically

This widget will automatically enrol a user for Bill Pay if they are not currently enrolled.

Enrolment status and enrolment is performed using: Adopt Bill Pay with the REST API

Features

This widget has the following features:

  • Checks current enrolment status

  • If not enrolled, will enrol that user for Bill Pay

  • Once complete, will automatically forward the user to their intended page, or the default page.

  • If enrolment fails, a retry button will be shown.

Related topics

For further information, see:

  • Reference - Enroll bill pay automatically

Enroll Payveris eBills

In this widget, customers enrol in eBills for a given payee. Customers enter the credentials for their payee account and, upon successful enrolment, receive billing information for that payee directly into their retail banking app. The customer enrols for each payee separately.

This widget interfaces with the Bill Pay DBS capability using the Bill Pay client API.

Features

This widget provides the following features for the customer:

  • Provide credentials for enrolment.

  • Enrol in multiple accounts from one payee.

  • Cancel enrolment.

  • Return to the payee list.

The widget provides additional authentication steps such as CAPTCHA and a one-time passcode for added security. The widget also handles error states forwarded from Payveris.
