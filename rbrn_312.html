Retail Banking Widget Collection v3.1.2

The highlights for this release are:

  • Card Management - new widget in Retail Banking Widget Collection.

  • User Manage Profile - new widget in Retail Banking Widget Collection.

  • Income and Spending Analysis per Category - new widget in Retail Banking Widget Collection.

  • Native Categories Widget - new widget in Retail Banking Widget Collection.

New features

The new features in this release are:

  • Card Management

    New widget: view the list of your credit/debit cards. You can temporarily lock/unlock any of them.

  • User Manage Profile

    New widget: manage your email address, postal address, and phone number.

  • Income and Spending Analysis per Category

    New widget: view a breakdown of income or spending per category for selected products and period.

  • Initiate SEPA Payment

    You can now schedule a recurring payment on bi-weekly basis.

  • Initiate SEPA Payment

    You can now save the beneficiary as a new contact.

  • Product Summary List

    Updated extension slots.

  • Product Summary Account Selector

    Updated extension slots.

  • Product Summary View Details

    Updated extension slots.

  • Categories

    New native widget to enable the use of categories.

Bug fixes

The following issues have been fixed in this release:

  • MAINT-6244 - Accounts with no products show in Account Dashboard

  • MAINT-6356 - No HTML slot for error handling

Deprecations

The following features are deprecated from this release. Support for these features ceases in v3.2.0.

  • Web widgets

    • The extension slots of the Transactions List widget listed below:

      • bbTransactionsListGroupCustomizable - use bbTransactionsListItemGroupCustomizable instead to target the component instead of the outer container.

      • bbTransactionsListItemDetailsCustomizable - use bbTransactionsListItemDetailsCustomizable2 to target the details or bbTransactionsListItemCustomizable to target the entire item (that is, header and details).

      • bbTransactionsListItemHeaderCustomizable - use bbTransactionsListItemHeaderCustomizable2 to target the component instead of the outer container.

      • bbTransactionsListListCustomizable - use bbTransactionsListListCustomizable2 to target the component instead of the outer container.

      • bbTransactionsListSearchFilterCustomizable - use bbTransactionsListControlsCustomizable instead to target the component instead of the outer container.

    • The following extension slots of the Transactions List widget will be removed in v3.2.0:

      • bbTransactionsListGroupCustomizable

      • bbTransactionsListSearchFilterCustomizable

    • The following extension slots of the Transactions List widget will be renamed in v3.2.0:

      • bbTransactionsListItemDetailsCustomizable2 will be renamed to bbTransactionsListItemDetailsCustomizable.

      • bbTransactionsListItemHeaderCustomizable2 will be renamed to bbTransactionsListItemHeaderCustomizable.

      • bbTransactionsListListCustomizable2 will be renamed to bbTransactionsListListCustomizable.

  • Native mobile widgets

    • Payments Client

      Android

      The entire Payments Client module is being deprecated and moved to a new Payment Order Client. The pre-existing endpoints continue to function as it was in the old client with the following name changes and new additions:

      • The createPayment() method changes to the createPaymentOrder() method.

      • A new getPaymentOrder() method with respective DTOs.

      • The PaymentResponse changes to the PaymentOrdersPostResponse with a few new additional fields.

      • The InstructedAmount property changes to the Amount object from common-utils.

      • The Creditor changes to InvolvedParty.

      • The CreditTransferTransactionInformation changes to InitiateCreditTransaction.

      iOS

      The entire Payments Client module is being deprecated and moved to a new Payment Order Client. The new client has a different implementation to align closely with the Android version.

    • Notifications Client

      The UpdateNotificationListener#onSuccess() and #onError(Response) methods are deprecated. Use onSuccess(String) and onError(String, Response) to receive the updated notification’s ID in each callback.

Breaking Changes

The following breaking changes have been introduced in this release:

The Native Transactions Widget is now publishing the events with specific modes.

  • Use 'overview' mode for following pub/sub events:

    • bb.action.product-summary.product-selected

    • TransactionsWidget.transactionSelectedAction

    • TransactionsWidget.transactionSelectedEvent

    • TransactionsWidget.showSearchListEvent

    • TransactionsWidget.hideSearchListEvent

    • TransactionsWidget.searchQueryChangedEvent

  • Use 'details' mode for

    • TransactionsWidget.transactionReceivedEvent pub/sub event.

Places v3.1.6

This release note explains the changes to this capability.

Bug fixes

The following issues have been fixed in this patch:

  • Native mobile

    • MAINT-8626: App Store rejection due to use of non-public API in Places widget for iOS.

      Use v3.1.6 of the Places widget. For the rest of the widgets in the collection, use v3.1.2.

      For example:

      pod 'PlacesWidget', '3.1.6'
pod 'TransactionsWidget', '3.1.2'

      For details on how to migrate Places to v3.1.6, visit the Community.
