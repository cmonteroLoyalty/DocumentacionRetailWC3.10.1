Account Statements v3.5.3

This release note explains changes to this capability.

Bug fixes

This release includes the following bug fixes:

  • MAINT-8790: Allow the file name to be set when downloading files

    In the account-statement-download-doc directive, the downloaded file had no name.

Authentication v3.5.3.1

This release note explains changes to this capability.

New features

  • Improved widget title

    To make its purpose clearer and improve consistency among widget collections, the Login widget has been retitled to Log in user widget.

Bug fixes

The following issues have been fixed in this patch:

  • MAINT-9389: Unable to use custom provider tokens for the Log in user widget

    To resolve this, providers are now added at the module level instead of the component level.

  • MAINT-9415: Users could only log in on the second attempt

    To resolve this, the Log in user widget has been updated so that the login post request now includes the responseType set to text.

To update to v3.5.3-sp.1, run the following command from your project application directory:

npm i --save @backbase/retail-ang@3.5.3-sp.1

Authentication v3.5.3

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Config providers, services, and components are now exported from the public API

    Configuration providers, services, and components are now exported from the Login widget public API. Including these elements in the API ensures that they are publicly accessible.

Bill Pay v3.5.3

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Improved widget navigation

    • The Bill Pay Payee Summary widget now navigates to the Bill Pay Manage Payee widget when no payees exist.

    • Cancel buttons have been added to the Bill Pay Manage Payee widget’s manual payee, electronic payee, and search payee screens. Clicking Cancel returns the user to the payee list screen.

    • Navigation handling has been standardised throughout the widgets.

  • Bill Pay Payee Summary text improvements

    The Bill Pay Payee Summary widget has the following text improvements:

    • The text for payee stepper has changed from 1 of 3 Payees to Payee 1 of 3.

    • The text for error and empty states now refers to support instead of system administrator.

  • Native Bill Pay widget now sends an event when payee with eBills enabled is selected

    The native Bill Pay widget now publishes a new event, bb.action.bill-pay.pay-payee-with-active-ebill. This is sent on selection of a payee with eBills enabled. The event’s payload contains a Payee Summary object. The native Bill Pay Form widget subscribes to this event.

Bug fixes

This release includes the following bug fix:

  • OBPAY-2290: Extra eBill icon displayed when SSO enabled

    The Bill Pay Payee List widget displayed an additional eBill icon when the backbase.payveris.sso property was enabled.

Payments v3.5.3

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Initiate an external A2A transfer (web)

    Users can now transfer funds to their own accounts at another bank. A prerequisite is that the user first connects and activates the external account. If you’re using DBS, note that DBS does not yet support initiating payments to external accounts. Banks need to implement their own custom solution to provide A2A transfers to their customers.

  • Omni-payment widget (web)

    The Omni-payment widget is a base widget that is fully customizable with a TypeScript configuration. The Omni-payment widget always requires a developer to either configure an out-of-the-box payment type, or to create and configure a custom payment type. For more information, see:

    • Configure an out-of-the-box payment type (beta)

    • Configure a custom payment type (beta)

    Note
    		 This is a beta (sample code) release. This feature may, unless otherwise authorized by Backbase, be used for testing purposes only, and is provided on an "AS IS" basis without support of any kind. You should not use this feature in a production environment.

Products v3.5.3

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Updated UI component

    All Products widgets are now updated to use bb-loading-indicator-ui instead of the deprecated bb-loading-state-ui:

    • Product Summary List

    • Product Summary Account Selector

    • Product Summary View Details

    • Product Summary Manage Accounts

  • BBAN account number is added to Term Deposit account

    BBAN account number is now added to TermDeposit account in the Android Products client.

Bug fixes

This release includes the following bug fixes:

  • MAINT-9039: Common business and retail banking ui-ang library @backbase/product-summary-common-ang was missing from Retail ui-ang dependencies.

    When using Widget Collection Retail Banking only, there was a compilation error in the application. The common library @backbase/product-summary-common-ang is now added to retail-ang/package.json as a dependency of collections-banking-ang to eliminate this issue.

  • MAINT-8873: Customization for bb-product-summary-account-selector-widget replaced with an out-of-the-box template when bank customer is navigating in Product Summary widgets

    Preferences for bb-product-summary-account-selector-widget were reset to the out-of-the-box default templates when a bank user was navigating in between Account Details, My Accounts, and Accounts list. This issue is now fixed and customizations are no longer reset to default templates on user navigation.

  • MAINT-9016: - Product Summary widgets using deprecated UI component

    All Products widgets are now updated to use bb-loading-indicator-ui instead of the deprecated bb-loading-state-ui:

    • Product Summary List

    • Product Summary Account Selector

    • Product Summary View Details

    • Product Summary Manage Accounts

Self Enrollment v3.5.3

This release note explains changes to this capability.

New features

This release provides the following new features:

  • New feature service for Theme 2.0 support

    To support Backbase Theme 2.0, a feature service has been added to the common library for the Self Enrollment widgets. This new service determines whether Theme 2.0 is enabled.

Transactions v3.5.3

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Ability to add a note to a transaction (native mobile)

    The native mobile Transactions now provides the ability to add, edit and delete notes for transactions.

  • Ability to view the geolocation and address of a transaction (native mobile)

    The native mobile Transactions now provides the ability to view the geolocation of a transactions in a map along with the open address so that users of banks can easily identify their transactions.

  • Ability to view the running balance in transactions (iOS native mobile)

    The native mobile Transactions now provides the ability view the running balance - the balance recorded after a transaction took place.

Bug fixes

This release includes the following bug fixes:

  • MAINT-8919: Extending bbTransactionsListItemGroupCustomizable breaks the page. Extension slot has been updated to not cause the breaking.
