Bill Pay v3.3.0

New features

  • Web widgets

    The billpay-payments-widget-ang widget has the following new features:

    • View payment details

    • Cancel payment

    • Account masking

    The billpay-manage-payment-widget-ang widget has the following new feature:

    • Account masking

    The billpay-payee-list-widget-ang widget has the following new feature:

    • Delete payee

  • Native mobile widgets

    • Manage bill payments: Electronic Payee

      There is now a feature to create a non-electronic payee.

Bug fixes

  • Native mobile widgets

    • Bill Pay Widget:

      • Android:

        • You can now edit a pending OVERNIGHT_CHECK payment.

Deprecations

  • Native mobile client

    • Common:

      • PayeeSearchResponse class is deprecated, and will be converted to a private class.

      • searchPayeeByName method that returns a PayeeSearchResponse object is deprecated. Instead use the searchPayeeByName method that returns a list of PayeeSearch objects.

    • Android:

      • Payment class - paymentAccountNumber is deprecated. Instead, use paymentAccount.paymentAccountNumber.

      • Payment class - paymentAccountNickName is deprecated. Instead, use paymentAccount.paymentAccountNickName.

      • The old Payment constructor has been deprecated and will be replaced with a new constructor containing the following additional parameters:

        • Payment.paymentAccount

        • Payment.payeeID.

      • The old Payee constructor has been deprecated and will be replaced with a new constructor containing the following additional parameter: Payee.paymentServices.

      • The old ElectronicPayee constructor has been deprecated and will be replaced with a new constructor containing the following additional parameter: ElectronicPayee.paymentServices.

    • iOS:

      • Payment class - paymentAccountNumber is deprecated. Instead, use paymentAccount.accountNumber.

      • Payment class - paymentAccountNickName is deprecated. Instead, use paymentAccount.accountNickName.

      • Payment class - paymentAccount and payeeId fields are now mandatory.

      • Payee class - paymentServices field is mandatory in the response.

      • The old initializers for Payment and Payee have been deprecated and will be replaced with new ones that include every mandatory property as an argument.

  • Native mobile widgets

    • Bill Pay Widget:

      • Common

        • bb.action.bill-pay.payee-selected event is deprecated. Instead, use bb.action.bill-pay.payeesummary-selected event.

    • Bill Pay Form Widget:

      • iOS

        • confirm mode is deprecated. Instead, use overview mode.

        • The default, initial mode is now initiate. Use initiate for for the initial mode.

        • We now support three modes: initiate, review and overview.

Breaking changes

  • Native mobile client

    • Android:

      • The Amount DTO class internal to this client is replaced with the Amount DTO class in CommonUtils.

    • iOS:

      • The BBCurrency DTO class internal to this client is replaced with the BBCurrency DTO class in CommonUtils.

Card Management v3.3.0

New features

The new features in this release are:

  • Web widgets

    • Users can activate an inactive debit/credit card by authenticating with the CVV code.

    • Users can track the delivery status of their cards.

  • Native mobile widgets

    • Users can activate an inactive debit/credit card by authenticating with the CVV code.

    • Users can track the delivery status of their cards.

Contacts v3.3.0.1

The highlights of this release are:

  • You can now customize the Contact Manager widget using additions.

    To update your Retail Banking Widget Collection to v3.3.0-sp.1, run the following command from your project application directory:

    npm i --save @backbase/retail-ang@3.3.0-sp.1

Bug fixes

The following bugs have been fixed in this release:

  • MAINT-7392 - When using the additions field while creating a new contact or updating an existing one, the field value is sent to the service as an empty object. It is not possible to add additions to the template.

Contacts v3.3.0

Bug fixes

  • Native mobile widgets:

    • Contacts Widget

      • Android: Client cache is no longer persisted in saveInstanceState.

Deprecations

  • Native mobile widgets:

    • Contacts Widget

      • Android

        • EVENT_DELETE_CONTACT will be removed. Instead use #deleteContact(Contact, CrudContactListener).

        • EVENT_CONTACT_RECEIVED is deprecated. Define navigation events in your app’s model with a navigation event of EVENT_CONTACT_SELECTED.

      • iOS

        • contactReceivedEvent() is deprecated. Define custom navigation in your app’s model.

        • deleteContactAction() is deprecated. Use deleteContact() method.

  • Native mobile clients:

    • Contacts Client

      • iOS

        • ContactItem method isEqualToContactItem: is replaced with isEqualTo:

        • ContactsClient method createContact:completionHandler: is replaced with createWithContact:completionHandler:

        • ContactItem (now BBBaseContact) property accessContextScope is deprecated.

Support for these features ceases in Contacts v3.5.0.

Messages v3.3.0

The highlights for this release are:

  • The unread conversation counter now uses publish-subscribe messaging.

  • Beta release of employee facing widgets.

New features

The new features in this release are:

  • The unread conversation counter now uses publish-subscribe messaging

    Client facing widgets now emit a frontend event which is picked up by the unread conversation counter.

  • Beta release of employee facing widgets

    These new widgets enable an employee to perform the following actions:

    • View a paginated list of conversations.

    • Navigate through the list using pagination.

    • See if a conversation is assigned to another employee.

    • Manually refresh the list.

    • View a conversation thread.

    • Assign a conversation to themself, and then unassign the conversation.

    • Reply to the conversation.

Notifications v3.3.0

Breaking changes

  • Native mobile client

    • Notifications

      • The createError method and its overloads were public static. It is now an instance method, and is not accessible statically.

      • The buildQuery method is removed from NotificationsDataImpl. It is now a part of its superclass DBSDataImpl.

Payments v3.3.0

Bug fixes

The following issues have been fixed in this release:

  • Native mobile widgets:

    • Payment Order widget

      • Android: Schedule object now gets updated when a schedule is received in ScheduleReceiver.

      • iOS: PaymentOrderScheduleWidget was not imported with PaymentOrderWidget; this is now imported with PaymentOrderWidget. The PaymentOrderScheduleWidget can now be implemented in a mobile application.

Deprecations

The following deprecations have been introduced in this release:

  • Native mobile widgets:

    • Payment Order Widget

      • Common

        • Autocomplete methods and properties are deprecated. Use your own method or navigation event name to show/dismiss the autocomplete.

        • Static properties that define the navigation event names are deprecated. Instead, use the following static methods:

      • Android

        • showAutocomplete is now EVENT_SHOW_AUTO_COMPLETE

        • dismissAutocomplete is now`EVENT_DISMISS_AUTO_COMPLETE`

        • EVENT_SELECT_PAYMENT_TYPE is now getSelectProductEventName(navigationInfix)

        • EVENT_SELECT_PRODUCT is now getSelectProductEventName(navigationInfix)

        • EVENT_SELECT_RECIPIENT is now getSelectRecipientEventName(navigationInfix)

        • EVENT_REVIEW_PAYMENT_ORDER is now getReviewPaymentOrderEventName(navigationInfix)

        • EVENT_SELECT_SCHEDULE is now getSelectScheduleEventName(navigationInfix)

        • EVENT_OVERVIEW_PAYMENT_ORDER is now getOverviewPaymentOrderEventName(navigationInfix)

        • EVENT_PRODUCT_SELECTED is now getProductSelectedEventName(navigationInfix)

        • EVENT_RECIPIENT_SELECTED is now getRecipientSelectedEventName(navigationInfix)

        • EVENT_SCHEDULE_CHANGED is now getScheduleChangedEventName(navigationInfix)

        • An ibanCodes.json file is no longer required to be included in your application’s assets folder, the widget will function properly with or without this file present.

      • iOS

        • showAutocomplete is now paymentOrderShowAutocompleteEvent

        • dismissAutocomplete is now paymentOrderDismissAutocompleteEvent

        • paymentOrderSelectProductAction is now paymentOrderSelectProductAction:infix

        • paymentOrderProductSelectedEvent is now paymentOrderProductSelectedEvent:infix

        • paymentOrderSelectRecipientAction is now paymentOrderSelectRecipientAction:infix

        • paymentOrderRecipientSelectedEvent is now paymentOrderRecipientSelectedEvent:infix

        • paymentOrderScheduleChangeRequest is now paymentOrderScheduleChangeRequest:infix

        • paymentOrderScheduleChangedEvent is now paymentOrderScheduleChangedEvent:infix

        • paymentOrderSelectPaymentTypeAction is now paymentOrderSelectPaymentTypeAction:infix

        • paymentOrderReviewAction is now paymentOrderReviewAction:infix

        • paymentOrderOverviewAction is now paymentOrderOverviewAction:infix

  • Native mobile clients:

    • Payment Order Client

      • Android

        • The default constructor for the Schedule class is deprecated and replaced with Schedule(TransferFrequency, Integer, Date, Integer).

Support for these features ceases in Payments v3.5.0.

Breaking changes

The following breaking change has been introduced in this release:

  • Web widgets:

    • The Initiate SEPA Payment widget payord-initiate-sepa-payment-widget has been removed. It is replaced by the more generic and configurable Initiate Payment widget payord-initiate-payment-widget.

Products v3.3.0

The highlights for this release are:

  • Display list of account owner details.

New features

  • Web The new features in this release are:

  • Display a list of account owner details in retail banking widgets.

    The List widget now contains the following details for one or more arrangement owners:

    • first name

    • last name

    The widget retrieves this information through a direct integration service. It does not require DBS.

  • Native mobile clients

    • Product Summary client:

      • Product DTO has new fields for addresses and debit cards.

      • ProductKind DTO now extends Additions.

    • Accounts client:

      • New fields added to ArrangementItem and DebitCard.

      • DTOs with date-type fields now offer getters returning a Date object.

  • Native mobile widgets

Deprecations

  • Native mobile clients

    • Product Summary client - in the Product class:

      • alias property is deprecated, instead use bankAlias property.

      • accountHolderAddressLine3 property is deprecated, instead use accountHolderStreetName property.

      • accountHolderAddressLine4 property is deprecated, instead use town property.

      • Android:

        • DebitCardItem class is deprecated and will be replaced with DebitCardItem from common utils. Until DebitCardItem is removed make use of following methods: In DebitCardItem class, convertToNewDebitCardItem and convertToOldDebitCardItem methods, and in Product class, use new getter and setter for debitCardsItems property.

      • iOS

        • DebitCardItem class is deprecated and will be replaced with BBDebitCardItem. Until DebitCardItem is removed, make use of following methods: In the DebitCardItem class, bbDebitCardItem and debitCardItem conversion methods, and in the Product class, use bbDebitCardsItems instead of debitCardsItems. This is an alternative property until the DebitCardItem class is removed.

    • Accounts client:

      • Android:

        • DebitCardItem class is deprecated and will be replaced by DebitCardItem class from Common Utils.

        • UserPreferencesItem class is deprecated and will be replaced by UserPreferencesItem class from Common Utils.

        • In ArrangementBaseItem class: debitCards property is deprecated due to DebitCardItem class deprecation. It will not be removed, but it will be converted to array of DebitCardItem from Common Utils and alias property is deprecated. Instead use bankAlias property. All of the property setters are deprecated.

        • In ArrangementItem class: productId property is deprecated. Instead use externalProductId property. legalEntitiesIds property is deprecated. Instead use legalEntityIds property. All of the property setters are deprecated.

        • updateUserPreferences method is deprecated, instead use updateUserPreferences method which has UserPreferencesItem from Common Utils.

        • Replace all DTO methods that accept or return a date in string form to accept or return a date in Date form instead.

        • Response-DTO setters are deprecated.

      • iOS:

        • All classes are deprecated and new ones are added with BB prefix. Booleans in new classes are converted to NSNumber to support nullability.

        • All client methods are deprecated and new ones are added that use BB prefixed new classes.

Breaking changes

  • Native mobile widgets

    • Product Summary widget:

      • Default views are removed from the widget and moved into the sample application source. All classes that had Default prefix are renamed without the Default prefix. For example, DefaultProductSummaryWidgetView is changed to ProductSummaryWidgetView.

  • Native mobile clients

    • Product Summary client:

      • String-based TotalBalance#aggregatedBalance field has been replaced with a BigDecimal field of the same name.

      • Removed ProductConverter#fillProductInfoForRecipientModel from the public API.

Remote Deposit Capture v3.3.0

Deprecations

The following features are deprecated from this release:

  • Native mobile widgets:

    • RDC Form Widget

      • Android

        • EVENT_SUBMIT_SUCCEEDED and EVENT_SUBMIT_FAILED will no longer be published automatically after a deposit is submitted. The app’s listener will still receive success or failure callbacks, and showSubmitSucceeded(RDCDeposit) or showSubmitFailed(List) can be called if these navigation events to be published.

        • Both submit_failed and submit_succeeded modes are deprecated. Use submitFailed and submitSucceeded instead.

      • iOS

        • dailyLimitDidLoad:(NSString *_Nullable)depositLimit withCurrency:(NSString *_Nullable)currencyString method is deprecated. Instead use dailyLimitDidLoad:(BBCurrency *_Nullable)limit.

        • Both succeeded and failed modes are deprecated. Use submitSucceeded and submitFailed instead.

        • The default, initial mode is now initiate. Use initiate for the first initial mode.

        • The supported modes are: initiate, review, submitSucceeded, submitFailed

Support for these features ceases in Remote Deposit Capture v3.5.0.

Transactions v3.3.0

Bug fixes

The following issues have been fixed in this release:

  • Native mobile widget

    • Transactions

      • Client cache is no longer persisted in saveInstanceState.

Cross-capability changes

This release note lists the changes for in this release that affect several or all capabilities.

Breaking changes

Native mobile widgets and clients

  • Android widgets and clients

    • All clients and widgets that had a dependency on Android support libraries now use AndroidX instead. For the majority of clients and widgets, this only affects @Nullable and @NonNull annotations.

    • All clients and widgets are now compiled with Java 8 source and target compatibility. Projects will need to be compiled with Java 8 to continue using clients and/or widgets.

  • Common Utils

    • In the BBUserPreferencesItem class, visible and favorite property types are converted to NSNumber to support nullability.
