Cross-capability changes

This release note lists the changes for in this release that affect several or all capabilities.

Important Change for Mobile Widgets

There is a new technical version number for the Native Mobile Widgets.

Until now, the Widget Collection has followed the version format: <release>.<major>.<minor>.<patch>. As of this major release, the <release> number has been removed from artifacts. This means that this release, instead of being Widget Collection 3.2.0(.0) will now be named Widget Collection 2.0.0 in the Backbase repo.

This change has been introduced because customers with patched versions of the collection (for example, 3.2.0.1) are currently unable to publish apps to the App store. This is due to the App Store Connect rules around apps with 4-digit version numbers.

The <release> number is now part of the repository URL, which is the version number developers add to their local CocoaPods and Gradle installations. To stay consistent, this will happen both for Android and iOS.

To use the new major version of the Widget Collection, update the Gradle and Cocoapods dependencies to reflect the repository name change.

It is important to understand the actual product version is still 3.2.0(.0).

End of support for iOS 9 and iOS 10

From this release, the Widget Collection will no longer support iOS 9 and 10. For more information, see Platform Support for Mobile.

Deprecations

The following features are deprecated from this release. Support for these features ceases in Retail Banking Widget Collection v3.3.0.

  • Native mobile widgets and clients

    • The common-UI library will be discontinued for both iOS and Android.

    • All clients and widgets with a dependency on Android support libraries will be updated to use AndroidX. For the majority of clients and widgets, this only affects @Nullable and @NonNull annotations. No deprecation will be added to client or widget code for this change.

    • All clients and widgets will be compiled with Java 8 source and target compatibility. Projects will need to be compiled with Java 8 to continue using clients and widgets. No deprecation will be added to client or widget code for this change.

    • Common-utils for Android

      • DBSDataImpl#buildGetRequest(URI, String, DBSDataProvider, Map) is deprecated in favor of DBSDataImpl#buildGetRequest(URI, String, Params, DBSDataProvider). Params is a map-like object that can be built in a type-safe manner.

      • GenericListener is deprecated in favor of ResponseBodyListener. The new interface’s signature and usage is the same, but the name makes its purpose clearer (in contrast to the new ResponseListener that returns the full Response object on success).

Bill Pay v3.2.0

This release note lists the changes for the Bill Pay capability in this release.

New features

The new features in this release are:

  • Simple payment.

  • Set recurring payment.

  • Add manual payee.

Deprecations

The following features are deprecated from this release. Support for these features ceases in Retail Banking Widget Collection v3.3.0.

  • Native mobile widgets

    • Bill Pay widget

      Android

      • bb.action.bill-pay.payee-selected event is deprecated. Use bb.action.bill-pay.payeesummary-selected instead.

      iOS

      • bb.action.bill-pay.payee-selected event is deprecated. Use bb.action.bill-pay.payeesummary-selected instead.

    • Bill Pay Form widget

      Android

      • BillPayFormView - updateViewWithPayee is deprecated, use updateViewWithPayeeSummary instead.

      iOS

      • The confirm mode is deprecated. Use the overview mode instead.

      • The supported modes are now: initiate, review, overview.

      • The default, initial mode is now initiate. Therefore, for the first initial mode, use the initiate mode.

      • BillPayFormView - updateViewWithPayee is deprecated, use updateViewWithPayeeSummary instead.

  • Native mobile clients

    • Bill Pay Client

      Android

      • Amount DTO class internal to this client will be replaced by the Amount DTO in common-utils.

      iOS

      • BBCurrency DTO class internal to this client would be replaced with the BBCurrency DTO in common-utils.

  • Native mobile clients

    • Bill Pay client

      Android

      • Amount DTO class internal to this client will be replaced by the Amount DTO in common-utils.

      iOS

      • BBCurrency DTO class internal to this client would be replaced with the BBCurrency DTO in common-utils.

Breaking Changes

The following breaking changes have been introduced in this release:

  • Native mobile clients

    • Bill Pay Client

      Android & iOS

      • GenericListener and methods that accept it have been replaced with method-specific listeners. The versions of each method with specific listeners are available now.

      • PaymentService:

        • paymentFee field: fee and type has changed from double to Amount.

        • get/setPaymentFee(double paymentFee) has changed to get/setFee(Amount fee).

      • RecurringPayment

        • paymentAmount field has changed to Amount and type has changed from Number to Amount. New constructor has been added to accept an Amount object for amount instead of Number as paymentAmount

        • getPaymentAmount() has changed to getAmount().

      • PaymentRequest

        • Amount field type has changed from double to Amount and the paymentAmount field has been removed. A new constructor has been added to accept an Amount object for paymentAmount instead of Number as amount.

        • get/setAmount(double amount) has been changed to get/setPaymentAmount(Amount amount).

      • PaymentAccount

        • balance field has been changed to accountBalance and type has changed from double to Amount.

        • get/setBalance(double balance) has been changed to get/setAccountBalance(Amount accountBalance).

      • Payment

        • paymentFee field has been changed to Fee and type has changed from double to Amount.

        • paymentAmount field has been changed to Amount and type has changed from double to Amount.

        • new constructor has been added to accept an Amount object for amount instead of Number as paymentAmount.

        • get/setPaymentAmount(double paymentAmount) has changed to get/setAmount(Amount amount).

        • get/setPaymentFee(double paymentFee) has changed to get/setFee(Amount fee).

      • NextPayment

        • paymentFee field has changed to Fee and type has changed from double to Amount.

        • paymentAmount field has changed to Amount and type has changed from double to Amount.

        • A new constructor has been added to accept an Amount object for amount instead of Number as paymentAmount.

        • get/setPaymentAmount(double paymentAmount) has changed to get/setAmount(Amount amount).

        • get/setPaymentFee(double paymentFee) has changed to get/setFee(Amount fee).

      • LatestBill

        • paymentAmount field has changed to Amount and type has changed from double to Amount.

        • A new constructor has been added to accept an Amount object for amount instead of Number as paymentAmount.

        • get/setPaymentAmount(double paymentAmount) has changed to get/setAmount(Amount amount).

  • Web widgets

    • Users can now request a replacement of their stolen, broken and lost cards.

  • Native mobile widgets

    • Users can now request a replacement of their stolen, broken and lost cards.

Known issues

The known issues in this release are:

  • Web widgets

    • TRANS-2008 - conflicts between customization slots when a widget is rendered more than once. When a user goes to details, and then back to the overview using the back button in the browser, cards are not clickable.

      Work around:

      Enable ENFORCE_WIDGET_WRAPPER to true. Also all custom templates to be moved out of the global registry (ie: moved out of the app.module) and placed into a "wrapper widget", as described in the Customize Widgets guide).

      Android

Remote Deposit Capture v3.2.0

Deprecations

The following features are deprecated from this release. Support for these features ceases in Retail Banking Widget Collection v3.3.0.

  • Native mobile widgets

    • RDC Form widget

      Android

      • EVENT_SUBMIT_SUCCEEDED and EVENT_SUBMIT_FAILED will no longer be published automatically after a deposit is submitted. The apps listener will still receive success or failure callbacks, and the app can call showSubmitSucceeded(RDCDeposit) or showSubmitFailed(List) if it wants these navigation events to be published.

      • Both submit_failed and submit_succeeded modes are deprecated. Instead, use submitFailed and submitSucceeded respectively.

      • readRdcFormDataFromStorage, readRdcDepositDataFromStorage and readValidationErrorsFromStorage methods are deprecated.

      iOS

      • Both succeeded and failed modes are deprecated. Instead, use submitFailed and submitSucceeded respectively.

      • The supported modes are now: initiate, review, submitSucceeded, submitFailed.

      • The default, initial mode is now initiate. Therefore, fot the first initial mode, use initiate.

      • dailyLimitDidLoad:(NSString *_Nullable)depositLimit with Currency:(NSString *_Nullable)currencyString method is deprecated. Instead, use dailyLimitDidLoad:(BBCurrency *_Nullable)limit.

Breaking Changes

The following breaking changes have been introduced in this release:

  • Native mobile clients

    • RDC Client

      Android

      • getRDCDailyLimit method with rdcDepositDailyLimitListener is deprecated. Use getRDCDailyLimit method with rdcDailyLimitListener instead.

      • RDCDailyLimit class is deprecated, use Amount class from CommonUtils instead.

      • AggregatedBalance class is deprecated, use Amount class instead.

      • RDCDeposit - amount and currency properties are deprecated, use depositAmount property which has type as Amount instead.

      • RDCFormData - amount and currency properties are deprecated, use formDataAmount property which is a Amount type instead.

      • RDCProduct - balance, availableBalance and currency properties are deprecated, use productBalance and productAvailableBalance properties which are Amount type instead.

      • RDCProductKind - Property type of aggregatedBalance is deprecated and will be changed to Amount type. As an alternative use Amount conversion methods in the AggregatedBalance class.

      iOS

      • loadRDCDailyLimitWithHandler method is deprecated, use loadRDCDailyLimit method instead.

      • AggregatedBalance class is deprecated, use BBCurrency from CommonUtils instead.

      • RDCDeposit - amount and currency properties are deprecated, use depositAmount property which is a BBCurrency type from CommonUtils instead.

      • RDCFormData - amount and currency properties are deprecated, use formDataAmount property which is a BBCurrency type from CommonUtils instead.

      • RDCProduct - balance, availableBalance and currency properties are deprecated, use productBalance and productAvailableBalance properties which are BBCurrency type instead.

      • RDCProductKind = Property type of aggregatedBalance is deprecated and will be changed to BBCurrency type. As an alternative use BBCurrency conversion methods in the AggregatedBalance class. :rootdir: ../../../..

      • TransactionItem(String,String,String,TypeGroup,Type,String,CreditDebitIndicator) constructor in TransactionItem is deprecated. Instead, use TransactionItem(String,String,String,TypeGroup,String,String,CreditDebitIndicator) constructor.

      • getType method in TransactionItem is deprecated and will be replaced with a method that returns a string.

      • setType(Type) method in TransactionItem is deprecated. Instead, use setType(string).

      • getTypeAsString() method in TransactionItem is deprecated and will be renamed to getType after the deprecation is finished.

      • Load method is deprecated, instead use getTransactions.

      iOS

      • Load method is deprecated, instead use retrieveTransactionsWithParams.
