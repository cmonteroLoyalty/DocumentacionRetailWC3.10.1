Authentication v3.6.1

This release note explains changes to this capability.

Bug fixes

This release includes the following bug fixes:

  • MAINT-9389: Unable to use custom provider tokens for the Log in user widget

    To resolve this issue, providers are now added at the module level instead of the component level.

Bill Pay v3.6.1

This release note explains changes to this capability.

Bug fixes

This release includes the following bug fixes:

  • KB-10: WC3 Theme 2 - eBills enrolment - 2 notification pop ups are shown when enabling eBills

    When enrolling to Payveris eBills, after enabling eBills, two notification pop-ups were displayed instead of one. This is now fixed to display only one notification pop-up.

Notifications v3.6.1

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Web: Notification origin icon

    Notifications in the Manage and count notifications and View sticky and pop-up notifications widgets now include an icon that indicates from which functionality they originated. For example, Messages, Payments, or Transactions.

  • Native mobile: Routing added to the Notifications client

    With routing added, the customer can navigate from a notification to the relevant widget or a target page inside a widget.

Payments v3.6.1

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Web: Withdraw from a connected external account.

    In addition to paying into an external account, the Initiate payment widget now allows the bank customer to withdraw funds from their connected external account.

  • Native mobile: Transfer money to a connected external account

    The native Create payment order widget now supports the EXTERNAL_A2A payment type with the USD currency to allow bank customers to transfer money to a connected external account. To support this, the native Payment order client now includes the EXTERNAL_ID parameter. See the widget reference documentation for details.

  • Native mobile: Search contacts in a recipient list

    The native selector widget now supports the ability to search for a contact in the recipient list.

Known issues

This release has the following known issues:

  • Discovered in v3.6.0

Known issues

This release has the following known issues:

  • Discovered in v3.6.0

    • MAINT-9720: Cannot have two omni-payment widgets with different payment types.

      When two or more widgets customize the Initiate omni-payment widget and use the withConfig method to provide the payment configuration, the second or last configuration made overrides all the previous configurations.

      To work around this issue, provide the paymentConfig and the business function as inputs to the Initiate omni-payment widget component, in the wrapper widget’s template. Do not use the withConfig method. This issue is fixed with the 3.6.2 release.

Transactions v3.6.1

This release note explains changes to this capability.

New features

This release provides the following new features:

  • Native mobile (Android): View the running balance in transactions

    The native mobile View transactions, turnovers, and spending/income analysis now provides the ability to view the running balance, that is, the balance recorded after a transaction took place.

  • Native mobile: Filter transactions by amount range, date range, and transaction type

    The following new widgets have been added to enable bank customers to find specific transactions within their transactions history by filtering on amount range, date range and transaction type:

    • Filter transactions by amount range

    • Filter transactions by date range

    • Filter transactions by transaction type

  • Native mobile: View check images in transaction details

    The View transaction details widget now enables bank customers to view images of both the front and the back of their cleared checks in transactions details.

  • Web: View check images in transaction details

    The View transactions list widget now enables bank customers to view images of both the front and the back of their cleared checks in transactions details.

Bug fixes

This release includes the following bug fixes:

  • MAINT-8877: All transactions are positive

    When extending the View transactions list widget, for all transactions the positive sign was displayed. This is now fixed to make transactions correctly reflect the sign.

Migration

There are no migration steps specific to this release.
